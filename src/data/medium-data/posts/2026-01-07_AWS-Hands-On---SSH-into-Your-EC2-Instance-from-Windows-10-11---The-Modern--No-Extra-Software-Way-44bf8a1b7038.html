Introduction

Good news for Windows 10 and 11 users: you no longer need PuTTY! Microsoft built OpenSSH right into Windows, so you can connect to your Linux EC2 instance using the exact same ssh command that Mac and Linux users enjoy. In this super-simple guide you’ll open PowerShell (or Command Prompt), fix the one-time key permissions issue Windows loves to throw, and be logged into your cloud server in under three minutes — all with tools that are already on your laptop.

Quick Checklist Before You Start

  • Windows 10 (build 1809+) or Windows 11
  • Your .pem key file downloaded (e.g., EC2Tutorial.pem)
  • EC2 instance running + public IP visible
  • Port 22 (SSH) allowed in the security group
  • Default Amazon Linux username = ec2-user

Step 1: Open PowerShell (or Command Prompt)

Press Win + X → choose Windows PowerShell or Terminal. (If ssh works here, you’re golden. If it says “not recognized”, fall back to the PuTTY method.)

Step 2: Navigate to Your Key Folder

cd $env:USERPROFILE\Desktop
# or wherever you saved the key
dir

You should see your EC2Tutorial.pem file.

Step 3: Connect (The Magic Command)

ssh -i EC2Tutorial.pem ec2-user@YOUR_PUBLIC_IP

Replace YOUR_PUBLIC_IP with the real one from the EC2 console.

First time? Type yes when it asks about the host fingerprint.

If You Get “Permissions are too open” (99% of Windows users hit this)

Windows is over-protective. Fix it in 30 seconds:

  1. Right-click your .pem file → Properties → Security tab → Advanced
  2. Click Disable inheritance → “Remove all inherited permissions”
  3. Click Add → Select a principal → Advanced → Find now
  4. Choose your user account → OK → check Full control → OK
  5. Apply → OK → OK

Now run the ssh command again — it works instantly and will never ask again!

You’re In! Test It

whoami          # → ec2-user
hostname        # → ip-something-ec2
ping google.com # → proves internet works (Ctrl+C to stop)

Bonus Tips

  • Public IP changes after stop/start → just copy the new one
  • Save the command in a .txt file or create a PowerShell alias so you never type it again
  • Works exactly the same in old Command Prompt if you prefer it

Conclusion

That’s literally it! No extra downloads, no key conversions, no PuTTY — just pure, clean SSH exactly like the pros use. You now have a direct line from your Windows laptop straight into the heart of your AWS server. From here you can install software, edit files, and build anything you want. Next lecture we’ll start doing real work inside your instance — see you on the command line! 🚀
