Question 1:

You’re getting errors while trying to create a new S3 bucket named “dev”. You’re using a new AWS Account with no S3 buckets created before. And you double-checked and found that you have the correct IAM permissions to create S3 Buckets. What is a possible cause for this?

Correct Answer : (2) S3 bucket names must be globally unique, meaning that once a name is taken by any user on AWS, it cannot be used again by anyone else, including you. In this case, the name “dev” is already in use, which is why you’re unable to create a bucket with that name.

Question 2:

You have enabled versioning in your S3 bucket which already contains a lot of files. Which version will the existing files have?

Correct Answer : (4) S3 bucket with versioning enabled, existing files do not have a version number assigned until they are modified, making their version effectively null. This understanding ties directly to knowing how versioning works in AWS S3.

Question 3:

You have updated an S3 bucket policy to allow IAM users to read/write files in the S3 bucket, but one of the users complain that he can’t perform a PutObject API call. What is a possible cause for this?

Correct Answer : (3) An explicit DENY takes precedence over any allowances made in the S3 bucket policy, meaning that if the IAM policy includes a DENY, it will override the permission granted by the bucket policy, preventing the user from performing the PutObject API call. Understanding this hierarchy of permissions is crucial for managing access in AWS effectively.

Question 4:

You want the content of an S3 bucket to be fully available in different AWS Regions. That will help your team perform data analysis at the lowest latency and cost possible. What S3 feature should you use?

Correct Answer : (4) It allows you to automatically copy objects from one S3 bucket to another, either in the same region or across different regions. This is essential for ensuring that your data is fully available for analysis with low latency and cost efficiency.

Question 5:

You have 3 S3 buckets. One source bucket A, and two destination buckets B and C in different AWS Regions. You want to replicate objects from bucket A to both bucket B and C. How would you achieve this?

Correct Answer : (1) This method allows you to utilize S3’s replication feature directly from the source bucket to each destination bucket, as S3 Replication does not support chaining. This ensures both destination buckets receive an independent copy of the objects from the source bucket.
