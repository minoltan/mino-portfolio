Here’s a step-by-step guide to playing with S3 versioning, based on the provided content:

Objective: Understand and practice Amazon S3 bucket versioning, including enabling versioning, uploading new file versions, rolling back to previous versions, and managing delete markers.

Prerequisites:

✔ An AWS account with access to Amazon S3.
✔ An S3 bucket.
✔ A basic HTML file (e.g., index.html) and an image file (e.g., coffee.jpg, beach.jpg) to upload.

Step 1: Upload Initial Files (Pre-Versioning)

  1. Go to “Objects”: Click on the “Objects” tab in your bucket.
  2. Upload beach.jpg and coffee.jpg: Upload beach.jpg and coffee.jpg to your bucket.
  3. Upload index.html (Initial Version): Upload your index.html file (e.g., with content "I love coffee").

Step 2: Enable Bucket Versioning

  1. Navigate to your S3 bucket: In the AWS S3 console, select the bucket you want to work with.
  2. Go to “Properties”: Click on the “Properties” tab for your bucket.
  3. Find “Bucket Versioning”: Locate the “Bucket Versioning” setting and click “Edit”.
  4. Enable Versioning: Select “Enable” and then click “Save changes”.
Explanation: Enabling versioning ensures that every time you upload a new version of an object, S3 keeps the previous versions instead of overwriting them.
Observation: At this point, beach.jpg, coffee.jpg, and index.html will have a null version ID because they were uploaded before versioning was enabled.

Step 3: Update a File and Observe New Versions

  1. Edit index.html: Open your index.html file locally and change its content (e.g., from "I love coffee" to "I really love coffee"). Save the changes.
  2. Upload the updated index.html: In your S3 bucket, click "Upload", add your modified index.html file, and proceed with the upload.
  • Verification: If you have your webpage hosted, refresh it to see “I really love coffee.”

3. Show Versions in S3:

  • In your S3 bucket’s “Objects” tab, look for the “Show versions” toggle (usually near the search bar or filter options). Enable it
Observation:
beach.jpg and coffee.jpg still have a null version ID.
index.html now has two versions:
One with a null version ID (the original upload before versioning was enabled).
One with a unique version ID (the newly uploaded “I really love coffee” version).
The version with the unique ID is the current version.

Step 4: Roll Back to a Previous Version (Permanent Delete of Current Version)

  1. Ensure “Show versions” is enabled.
  2. Identify the current version: Find the index.html object with the unique version ID (the one with "I really love coffee").
  3. Delete the current version: Click on the checkbox next to this specific version ID, then click “Delete”.
  4. Confirm permanent delete: In the confirmation dialog, type permanently delete in the text box and click "Delete objects".
Explanation: Deleting a specific version ID performs a “permanent delete” of that version. Since the latest version is removed, the previous version (the one with the null ID in this case) becomes the current version.
Verification: Refresh your webpage. It should now display “I love coffee.”

Step 5: Delete an Object and Observe a Delete Marker

  1. Disable “Show versions”: Toggle off “Show versions” in the S3 console.
  2. Delete coffee.jpg: Select the coffee.jpg object and click "Delete".
  3. Confirm delete: In the confirmation dialog, type delete (not permanently delete) in the text box and click "Delete objects".
Observation: From the main “Objects” view (with “Show versions” disabled), coffee.jpg appears to be gone.
Verification: If coffee.jpg was displayed on your webpage, force refresh (Ctrl+Shift+R or Cmd+Shift+R) your webpage. The image should no longer be visible, and attempting to open its direct URL will result in a "404 Not Found" error.

Step 6: Restore an Object by Deleting the Delete Marker

  1. Enable “Show versions”: Toggle on “Show versions” in the S3 console.
  2. Identify the Delete Marker: You will now see coffee.jpg listed with a "Delete marker" next to it and a unique version ID. The actual coffee.jpg file (with its original version ID) is still present underneath the delete marker.
  3. Delete the Delete Marker: Select the checkbox next to the “Delete marker” for coffee.jpg, then click "Delete".
  4. Confirm permanent delete: In the confirmation dialog, type permanently delete and click "Delete objects".
Explanation: Deleting the delete marker removes the “overwriting” effect of the marker, making the underlying object visible again.
Verification: Force refresh your webpage. The coffee.jpg image should now be visible.

Conclusion:

You have successfully experimented with S3 versioning, understanding how it allows for maintaining multiple versions of an object, rolling back to previous states, and the concept of delete markers when an object is “deleted” while versioning is enabled. This provides a robust mechanism for data recovery and change management within your S3 buckets.
