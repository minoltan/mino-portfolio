Stop Guessing — Here’s Exactly When to Use For vs While Loop

When I was a university student, it was 2 a.m. My assignment was due in 6 hours. I stared at my screen, fingers frozen. My code needed a loop — but which one? For? While? I randomly picked one, it broke. Switched to the other, still broke. I had no idea what I was doing — I was just guessing.
That night I discovered one simple question that made the choice obvious every single time. I never confused them again.

The One Question That Changes Everything

Before writing any loop, stop and ask yourself:

“Do I know how many times I need to loop?”

That’s it. That’s the whole secret.

  • Yes, I know → use for
  • No, I don’t know → use while

Everything else flows from this one question. Let me show you exactly what I mean.

The For Loop — When You Know the Count

After that rough night, my professor explained it to me with the simplest example I’d ever heard.

“Imagine you’re doing pushups, if your trainer says do exactly 10 pushups — you know the count before you even start. That’s a for loop.”

for(int i = 0; i < 10; i++){
    System.out.println("Pushup: " + i);
}

It clicked instantly. The for loop has three things built right into it — where to start, where to end, and how to step. You know all three before the loop even begins.

The most common place you’ll use this is going through an array — because you always know exactly how many elements are in it.

int[] grades = {80, 90, 70, 85, 95};

for(int i = 0; i < grades.length; i++){
    System.out.println("Grade: " + grades[i]);
}

You know you need to visit every grade from index 0 to the end. Crystal clear count. Perfect for loop situation.

The While Loop — When You Don’t Know the Count

A few weeks later I ran into a different problem. I was building a small program that asked users to enter a positive number. The problem was — I had no idea how many times the user would enter a wrong number before getting it right.

Maybe once. Maybe ten times. Maybe never.

“That’s a while loop”,
“Imagine eating until you’re full. You don’t know how many bites it’ll take before you even start — you just keep going until something changes.”

int input = -1;

while(input < 0){
    System.out.println("Enter a positive number: ");
    input = scanner.nextInt();
}

You don’t control the count — the condition controls when you stop. That’s the soul of a while loop.

Another classic example — two pointers moving toward each other:

int left = 0;
int right = nums.length - 1;

while(left < right){
    // compare and move
    left++;
    right--;
}

How many steps until left and right meet? You have no idea — it depends on the data. So you just keep going while the condition is true.

Four Scenarios — Let’s Make It Permanent

By the time my semester ended, I had seen enough loops to recognize the pattern instantly. Here are the four scenarios that cover almost every situation you’ll face:

Scenario 1 — Checking every element in an array

// "Check every student's grade"
int[] grades = {80, 90, 70, 85};

for(int i = 0; i < grades.length; i++){
    System.out.println(grades[i]);
}

You know exactly how many students there are. Use for.

Scenario 2— Repeat exactly N times

// "Send reminder email 3 times"
for(int i = 0; i < 3; i++){
    System.out.println("Sending reminder " + i);
}

Fixed count, known before starting. Use for.

Scenario 3— Keep going until something happens

// "Keep dividing until number reaches zero"
int n = 100;
while(n > 0){
    n = n / 2;
    System.out.println(n);
}

You don’t know how many divisions it takes. Use while.

Scenario 4 — Two pointers moving toward each other

int left = 0;
int right = nums.length - 1;

while(left < right){
    left++;
    right--;
}

Unknown number of steps until they meet. Use while.

Scenario 5 — Keep guessing until you get it right

Imagine a number guessing game. The player keeps guessing until they find the correct number. You have no idea if they’ll get it in 1 try or 100 tries.

int secret = 42;
int guess = -1;

while(guess != secret){
    System.out.println("Guess a number: ");
    guess = scanner.nextInt();
}

System.out.println("You got it!");

You don’t know how many wrong guesses the player will make — you just keep going until they get it right.

The Cheat Sheet

The Night Everything Changed

Looking back at that 2 a.m panic — I wasn’t struggling because loops were hard. I was struggling because nobody had given me the one question to ask before writing a loop.

Once I had that question, everything became obvious. I stopped guessing. I stopped randomly switching between for and while hoping something would work.

Now every time I see a loop problem I ask myself:

“Do I know how many times I need to loop?”

And the answer tells me exactly what to write.

You’ll feel it too — after enough practice, you won’t even need to ask the question. Your fingers will just know. That’s when you realize you’ve stopped being a beginner.

Start with the question. The feeling will follow.
