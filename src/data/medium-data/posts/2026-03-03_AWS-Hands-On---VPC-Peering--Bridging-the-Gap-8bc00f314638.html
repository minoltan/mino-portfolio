🌟 Introduction

VPC peering allows you to connect two Virtual Private Clouds as if they were part of the same network. Even if you have two separate VPCs with different IP ranges (CIDRs), a peering connection bridges them using private IP addresses. In this exercise, we demonstrate that VPCs are isolated by default. By creating a Peering Connection, accepting the handshake, and — most importantly — updating the Route Tables in both VPCs, we establish a two-way communication channel. This allows an instance in a default VPC to communicate with a web server in our custom Demo VPC using only private IPs.

Step-by-Step Guide: Peering Two VPCs

1. Launch Default EC2 Instance

2. Verification of Isolation

Before peering, confirm that the instances cannot “talk” to each other.

  • Instance A (Demo VPC): Has a private IP like 10.0.0.216.
  • Instance B (Default VPC): Has a private IP like 172.31.30.189.
  • The Test: Try to curl the private IP of Instance A from Instance B.
curl 10.0.0.72:80
  • Result: The request times out because the networks are isolated.

2. Create the Peering Connection

  • Navigate to VPC Dashboard > Peering Connections.
  • Click Create peering connection.
  • Name: demo-peering-connection.
  • Requester VPC: Select your Demo VPC.
  • Accepter VPC: Select the Default VPC.
  • Click Create.

3. Accept the Request

The connection stays in a “Pending Acceptance” state until the owner of the second VPC approves.

  • Select the new peering connection.
  • Go to Actions > Accept Request.
  • Once accepted, the status changes to Active.

4. Update Route Tables (Two-Way Street)

A peering connection exists, but the “traffic controllers” (Route Tables) don’t know about it yet. You must add routes in both directions.

A. Route from Demo VPC to Default VPC:

  • Go to the Public Route Table of the Demo VPC.
  • Edit Routes > Add Route.
  • Destination: Enter the CIDR of the Default VPC (e.g., 172.31.0.0/16).
  • Target: Select Peering Connection and pick your demo-peering-connection.

B. Route from Default VPC to Demo VPC:

  • Go to the Main Route Table of the Default VPC.
  • Edit Routes > Add Route.
  • Destination: Enter the CIDR of the Demo VPC (e.g., 10.0.0.0/16).
  • Target: Select Peering Connection and pick your demo-peering-connection.

5. Final Connection Test

Go back to your terminal on the Default VPC Instance.

  • Run the curl command again:
curl 10.0.0.72:80
  • Result: You should now see “Hello World!” returned immediately.

🧾 Conclusion

VPC Peering is a powerful tool for connecting resources across accounts or regions without the overhead of a VPN or a physical gateway. However, it is essential to remember that peering is non-transitive. If VPC A is peered with VPC B, and VPC B is peered with VPC C, VPC A cannot talk to VPC C through B. To build more complex, multi-VPC networks without a full mesh of peering links, you would eventually look into using a Transit Gateway, which acts as a central hub for all your VPC connections.
