Step 1: Open Kinesis Service

  1. Log in to the AWS Management Console.
  2. Navigate to Amazon Kinesis service.
  3. Select Data Streams (we are focusing on Data Streams, not Firehose or Data Analytics).

Step 2: Create a Data Stream

  1. Click on Create data stream.
  2. Provide a name, e.g., DemoStream.
  3. Choose the capacity mode:
  • On-demand (auto-scales, pay-per-throughput).
  • Provisioned (manually define number of shards; 1 shard = 1MB/s write, 2MB/s read).

4. For this demo, select 1 shard (cost-effective).

5. Click Create data stream and wait until the stream is active.

Step 3: Understand Producers & Consumers

  • Producers send data to the stream (using SDK, Kinesis Producer Library, or Kinesis Agents).
  • Consumers read data from the stream (using Lambda, Firehose, Kinesis Client Library, etc.).
  • In this demo, we’ll use AWS CLI to act as both producer and consumer.

Step 4: Open AWS CloudShell

  1. Click on the CloudShell icon (top-right, next to the bell).
  2. This opens a CLI environment in AWS (preconfigured with IAM permissions and region).
  3. Check your AWS CLI version:
aws --version

Step 5: Send (Produce) Data to Kinesis Stream

Use the put-record command:

aws kinesis put-record \
  --stream-name DemoStream \
  --partition-key user1 \
  --data "user signup" \
  --cli-binary-format raw-in-base64-out
  • Change DemoStream if you used a different name.
  • Run this multiple times with different data (user login, user logout, etc.).
  • Each record is stored in the stream with an ID and sequence number.

Step 6: Describe the Stream (to Get Shard Info)

aws kinesis describe-stream --stream-name DemoStream

Note the Shard ID (e.g., shardId-0000000000000) — required for consuming data.

Step 7: Read (Consume) Data from Kinesis Stream

First, get a Shard Iterator:

aws kinesis get-shard-iterator \
  --stream-name DemoStream \
  --shard-id shardId-0000000000000 \
  --shard-iterator-type TRIM_HORIZON

TRIM_HORIZON reads from the beginning of the stream.

Copy the value of ShardIterator from the output.

Step 8: Fetch Records Using ShardIterator

Use the retrieved iterator to get records:

aws kinesis get-records --shard-iterator <YourShardIterator>

You’ll receive records with PartitionKey, Timestamp, and Data (Base64 encoded).

Step 9: Decode the Base64 Data

  • Copy the encoded Data value.
  • Use an online tool (e.g., Base64 Decode) or command line:
echo "BASE64_STRING" | base64 --decode

You’ll see your original messages (e.g., user signup, user login).

🟢 Part 2 : Delivery Streams

Step 10: Practice with Kinesis Data Firehose

After successfully creating and testing your Kinesis Data Stream, the next step is to set up a Kinesis Data Firehose Delivery Stream that will deliver data from the stream to an Amazon S3 bucket.

Step 11: Create a Delivery Stream

  1. Navigate to the Kinesis service in the AWS Management Console.
  2. Click on Delivery Streams.
  3. Click Create delivery stream.
  4. Review the diagram showing how Firehose works:
  • Producers send data to Firehose (via Kinesis Data Stream, Direct PUT, AWS services, or SDK).
  • Optional transformation using AWS Lambda.
  • Data is delivered to destinations such as Amazon S3, Amazon OpenSearch Service, Amazon Redshift, or HTTP endpoints.

Step 12: Configure the Delivery Stream

Source:

  • Select Kinesis Data Stream as the source.
  • Choose the stream you created earlier (DemoStream).

Destination:

  • Choose Amazon S3 as the destination.
  • Select an existing bucket (or create a new one).
  • Example bucket name: demo-firehose-yourname.

Step 13: Transformation & Record Conversion (Optional)

  • You can enable transformation using Lambda to preprocess your data.
  • Optionally, you can convert record formats to Parquet or ORC.
  • For this demo, leave these disabled to keep things simple.

Step 14: Buffering, Compression, and Permissions

  • Buffer Size: Set to 1 MB (for faster delivery).
  • Buffer Interval: Set to 60 seconds (minimum).
  • Compression: You can enable GZIP to save space, but it’s optional.
  • Encryption: Optional — you can enable SSE (Server-Side Encryption).
  • IAM Role: Allow AWS to automatically create the required IAM role for Firehose to access your S3 bucket.

Step 15: Create the Delivery Stream

  • Click Create delivery stream.
  • Wait until the status shows Active.

🟢 Part 3 : Flow test

Send Data Through Kinesis Stream → Firehose → S3

Now that Firehose is set up, it will only process new data from the Kinesis Data Stream.

Step 16: Send Data Using AWS CLI (CloudShell)

  1. Open CloudShell in AWS (icon next to the bell).
  2. Run the following command to send data to your stream:
aws kinesis put-record \
  --stream-name DemoStream \
  --partition-key user1 \
  --data "user signup" \
  --cli-binary-format raw-in-base64-out

3. Send a few more records:

aws kinesis put-record --stream-name DemoStream --partition-key user1 --data "user login" --cli-binary-format raw-in-base64-out
aws kinesis put-record --stream-name DemoStream --partition-key user1 --data "user logout" --cli-binary-format raw-in-base64-out

Step 17: Verify Data in Amazon S3

  1. Go to Amazon S3 in the AWS console.
  2. Open the bucket used for Firehose (demo-firehose-yourname).
  3. Initially, you’ll see zero objects because Firehose buffers data for 60 seconds.
  4. Wait at least 60 seconds, then refresh the bucket.
  5. Open the file to see the records (user signup, user login, user logout) delivered successfully.

Step 18: Cleanup Resources

To avoid unnecessary costs:

  • Delete the Firehose Delivery Stream.
  • Delete the Kinesis Data Stream (DemoStream).
  • Optionally, delete the S3 bucket if no longer needed.
