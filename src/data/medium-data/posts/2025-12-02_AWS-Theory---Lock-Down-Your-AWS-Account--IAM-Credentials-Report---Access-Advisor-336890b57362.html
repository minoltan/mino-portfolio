Introduction

You’ve created users, groups, roles, and policies — but how do you know if your account is actually secure? Hardening IAM isn’t a one-time setup; it’s an ongoing process. Thankfully, AWS gives us two completely free, built-in security tools that do the heavy lifting: the IAM Credentials Report (account-wide hygiene check) and IAM Access Advisor (least-privilege enforcement). Together they answer the two most important questions every AWS administrator should ask regularly:

  1. “Are my credentials safe and rotated?”
  2. “Are we giving away more permissions than we actually use?” Let’s walk through both tools and turn them into a repeatable security habit.

1. IAM Credentials Report — Your Account-Wide Credential Health Check

This is a single CSV report that lists every IAM user in your account plus the status of every credential type:

How to generate it (30 seconds)

  1. IAM → Dashboard → Generate credentials report
  2. Wait a few seconds → Download CSV

Pro tip: Automate it monthly with a simple Lambda + EventBridge rule and email yourself the report — zero excuses.

2. IAM Access Advisor — The Least-Privilege Superpower

Access Advisor shows the last time each AWS service was actually used by a specific user, role, or policy. This is pure gold for removing bloat.

Example findings you’ll see all the time:

  • A developer has AmazonS3FullAccess but hasn’t touched S3 in 400 days
  • A Lambda execution role has rds-db:connect permission but the function never hits RDS
  • A user still has ec2:* from an experiment two years ago

How to use it

  1. IAM → Users → [username] → Access Advisor tab
  2. Or IAM → Roles → [rolename] → Access Advisor tab
  3. Sort by “Last accessed” → anything blank or >90 days is a candidate for removal

Real-world workflow that pays for itself in the first week:

  1. Generate Credentials Report → rotate old keys + enable MFA everywhere
  2. Open Access Advisor on every user/role → create a new tighter policy (or use AWS-managed “ReadOnly” versions)
  3. Replace the old bloated policy → repeat quarterly

Bonus: Put It on Autopilot (Best Practice)

# One-liner to download the latest report via CLI
aws iam generate-credential-report
aws iam get-credential-report - query 'Content' - output text | base64 -d > iam-credential-report-$(date +%Y-%m-%d).csv

Add that to a monthly CloudWatch Event → you’ll never forget again.

Conclusion

Security in AWS isn’t about doing one big audit and calling it done — it’s about building small, repeatable habits. The IAM Credentials Report and Access Advisor are the two sharpest tools in the box for exactly that. Ten minutes with these tools every quarter will catch dormant access keys, missing MFA, and permission creep before they ever become a breach. Run them today, fix what you find, schedule them to run forever — and sleep much better knowing your IAM house is truly in order. Your future incident-response team (or certification exam) will thank you.
