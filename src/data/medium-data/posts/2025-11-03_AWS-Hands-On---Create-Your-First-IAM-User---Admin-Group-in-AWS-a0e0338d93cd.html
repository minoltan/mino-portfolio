Introduction

Welcome to your very first hands-on AWS lab!

If you’re new to AWS, one of the most important security steps is stopping the use of the root user for daily tasks. The root user has unlimited power — it can do anything in your account, including accidentally delete everything or rack up huge bills.

In this step-by-step, beginner-friendly guide, you’ll learn how to:

  • Create a secure IAM user (your personal admin)
  • Set up an Admin group with full permissions
  • Attach AdministratorAccess using best practices
  • Create a custom sign-in URL (account alias)
  • Log in side-by-side with root and IAM user

By the end, you’ll be using AWS the right way — safely, securely, and professionally.

Prerequisites

  • An AWS account (free tier works!)
  • Logged in as root user
  • A modern browser (Chrome, Firefox, Safari)

Step 1: Open the IAM Console

  1. Log in to the AWS Management Console as root user
  2. In the top search bar, type IAM
  3. Click IAM under Services
Key Fact:
IAM is a global service — no region selection needed.
Your users work everywhere in your account.

Step 2: Create an Admin Group

Let’s create a group first — this is best practice.

  1. In the left sidebar, click User groups
  2. Click Create group
  3. Fill in: User group name: admin
  4. Scroll to Attach permissions policies
  5. Search: AdministratorAccess
  6. Check the box
  7. Click Create group
Why groups?
Attach a policy once to a group → all members get the same access.

Step 3: Create Your IAM User

Now create your secure login.

  1. Left sidebar → Users
  2. Click Create user
  3. User name: mino (or your name)
  4. Check: Provide user access to the AWS Management Console
  5. Select: I want to create an IAM user
  6. Console password:
  • Select Custom password
  • Enter: MySecureP@ssw0rd123! (use a strong one!)
  • Uncheck: “Require password reset”

7. Click Next

Step 4: Add User to Admin Group

  1. On Set permissions
  • Choose Add user to group
  • Check admin

2. Click Next

Step 5: (Optional) Add Tags

Tags help organize and track resources.

  1. Click Add tags
  2. Key: department | Value: engineering
  3. Click Next

Step 6: Review & Create

  1. Confirm:
  • User: mino
  • Group: admin
  • Policy: AdministratorAccess

2. Click Create user

Step 7: Set Up a Custom Sign-In URL (Alias)

Make login simpler and safer.

  1. Go to IAM Dashboard
  2. Find Sign-in URL for IAM users → click Customize
  3. Enter: aws-mino-v1 (must be unique)
  4. Click Create

Your new login URL:

Step 8: Log In as Your IAM User

Pro Tip: Use two windows to compare root vs IAM.

Private Window (Recommended)

  1. Open a new incognito/private window
  2. Go to: https://aws-mino-v1.signin.aws.amazon.com/console
  3. Select IAM user
  4. Enter:
  • Account ID or alias: aws-mino-v1
  • IAM user name: mino
  • Password: Your custom password

5. Click Sign in

Step 9: Verify IAM Login

Check top-right corner:

Step 10: Explore Permissions

  1. Go to Users → mino
  2. Click Permissions tab
  3. See: AdministratorAccess → from group admin
Best practice: Permissions come from groups, not directly attached.

Conclusion

Congratulations! You just took the most important security step in AWS:

You now:

  • Have a secure personal admin (mino)
  • Understand groups and policies
  • Use a custom, clean login URL
  • Follow AWS best practices

You’re no longer a beginner — you’re building AWS the right way.
