AWS Use Cases | Uploading Profile Images Securely to S3 via Lambda + API Gateway + CloudFront

Managing profile images for your application? Let’s walk through how to store them securely in Amazon S3 and ensure they’re only accessible via CloudFront, not directly from the S3 bucket. We’ll do everything using the AWS Console — no code required!

🎯 Goal:

✅ Set Up the S3 Bucket (Private)
✅ Create a CloudFront Distribution
✅ Secure the S3 Bucket with CloudFront (OAC)
✅ Create the Lambda Function for Uploading 
✅ Grant Lambda Permissions to Access S3
✅ Create an API Gateway REST API (with Authorizer)
✅ Test the Image Upload API

Architecture Diagram

🔒 1️⃣ Set Up the S3 Bucket (Private)

  1. Go to S3 in the AWS Console.
  2. Create a Bucket:
  • Name it (e.g., my-profile-images-bucket).
  • Block all public access (this is critical).

3. Bucket Policy:

  • Do NOT allow public access.

🌐 2️⃣ Create a CloudFront Distribution

  1. Go to CloudFront → Create Distribution.
  2. Origin Settings:
  • Origin Domain: Choose the S3 bucket endpoint (not the static website endpoint).
  • Origin Access Control (OAC): Create a new one.

3. Behaviors:

  • Cache Policy: Use the Managed-CachingOptimized.

4. Distribution Settings:

  • Enable HTTPS.
  • Add a custom domain if needed (optional).

5. After distribution is deployed, note down the CloudFront domain name (e.g., d1234abcd.cloudfront.net).

6. Keep Save this Json Policy to add in Lambda permission

{
        "Version": "2008-10-17",
        "Id": "PolicyForCloudFrontPrivateContent",
        "Statement": [
            {
                "Sid": "AllowCloudFrontServicePrincipal",
                "Effect": "Allow",
                "Principal": {
                    "Service": "cloudfront.amazonaws.com"
                },
                "Action": "s3:GetObject",
                "Resource": "arn:aws:s3:::imi-2.0-profile-images/*",
                "Condition": {
                    "StringEquals": {
                      "AWS:SourceArn": "arn:aws:cloudfront::123456789:distribution/E44332DRG3G3"
                    }
                }
            }
        ]
      }

🔐 3️⃣ Secure the S3 Bucket with CloudFront (OAC)

🧩 4️⃣ Create the Lambda Function for Uploading

  1. Go to Lambda → Create Function.
  2. Add an IAM Role with permissions:
  • s3:PutObject on your bucket.
  • To create the lambda function with this code structure — Hands-On (Will Update soon..)
  • We need to install needed dependency

3. Lambda Code (Node.js example):

import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
import Busboy from 'busboy';
import crypto from 'crypto';

const s3Client = new S3Client({ region: process.env.AWS_REGION });

export const handler = async (event) => {
  try {
    console.log('Event received:', JSON.stringify({ isBase64Encoded: event.isBase64Encoded, headers: event.headers }, null, 2));

    // Always treat body as binary when sending binary data
    const bodyBuffer = Buffer.from(event.body, event.isBase64Encoded ? 'base64' : 'binary');
    console.log('Body buffer length:', bodyBuffer.length);
    console.log('Body buffer first bytes:', bodyBuffer.slice(0, 16).toString('hex'));

    const { files, fields } = await parseMultipart(bodyBuffer, event.headers);

    if (!files?.length) {
      return respond(400, { error: 'No file uploaded' });
    }

    const file = files[0];
    const fileName = sanitizeFilename(file.filename);
    const key = `profiles/${fileName}`;

    // Optional: Hash the file for integrity check
    const hash = crypto.createHash('sha256').update(file.content).digest('hex');
    console.log(`File SHA256: ${hash}`);

    console.log('File info:', {
      originalName: file.filename,
      size: file.content.length,
      type: file.contentType,
      firstBytes: file.content.slice(0, 16).toString('hex'),
    });

    await s3Client.send(
      new PutObjectCommand({
        Bucket: process.env.S3_BUCKET_NAME,
        Key: key,
        Body: file.content,
        ContentType: file.contentType || getMimeType(fileName),
        Metadata: {
          'original-filename': fileName,
          'sha256': hash,
        },
      })
    );

    const fileUrl = `${process.env.CLOUDFRONT_URL}/${encodeURIComponent(key)}`;

    return respond(200, {
      url: fileUrl,
      filename: fileName,
    });

  } catch (error) {
    console.error('Upload error:', error);
    return respond(500, {
      error: 'Upload failed',
      details: error.message,
      stack: process.env.NODE_ENV === 'development' ? error.stack : undefined,
    });
  }
};

const parseMultipart = (bodyBuffer, headers) => {
  return new Promise((resolve, reject) => {
    const contentType = getContentType(headers);
    const busboy = Busboy({
      headers: { 'content-type': contentType },
      limits: { files: 1, fileSize: 10 * 1024 * 1024 }, // 10MB limit
    });

    const result = { files: [], fields: {} };

    busboy.on('file', (fieldname, file, info) => {
      const { filename, encoding, mimeType } = info;
      const chunks = [];

      file.on('data', (chunk) => chunks.push(chunk));
      file.on('end', () => {
        result.files.push({
          fieldname,
          filename,
          encoding,
          contentType: mimeType,
          content: Buffer.concat(chunks),
        });
      });
    });

    busboy.on('field', (fieldname, value) => {
      result.fields[fieldname] = value;
    });

    busboy.on('error', (error) => {
      console.error('Busboy error:', error);
      reject(error);
    });

    busboy.on('finish', () => resolve(result));

    busboy.end(bodyBuffer);
  });
};

const getContentType = (headers) => {
  const headerKeys = Object.keys(headers || {});
  const contentTypeKey = headerKeys.find((key) => key.toLowerCase() === 'content-type');
  return headers?.[contentTypeKey] || 'multipart/form-data';
};

const sanitizeFilename = (filename) => {
  const parts = filename.split('.');
  if (parts.length < 2) return filename.replace(/[^\w.-]/g, '').toLowerCase();
  const ext = parts.pop();
  const name = parts.join('.').replace(/[^\w.-]/g, '').replace(/\s+/g, '-').toLowerCase();
  return `${name}.${ext}`;
};

const getMimeType = (filename) => {
  const ext = filename.split('.').pop().toLowerCase();
  const types = {
    jpg: 'image/jpeg',
    jpeg: 'image/jpeg',
    png: 'image/png',
    gif: 'image/gif',
    webp: 'image/webp',
    svg: 'image/svg+xml',
    bmp: 'image/bmp',
    pdf: 'application/pdf',
  };
  return types[ext] || 'application/octet-stream';
};

const respond = (statusCode, body) => ({
  statusCode,
  body: JSON.stringify(body),
  headers: {
    'Content-Type': 'application/json',
    'Access-Control-Allow-Origin': '*',
  },
});

🗝 5️⃣ Grant Lambda Permissions to Access S3

Attach this policy to your Lambda role:

{
  "Effect": "Allow",
  "Action": "s3:PutObject",
  "Resource": "arn:aws:s3:::my-profile-images-bucket/profiles/*"
}
  • The lambda should be configured with correct permission
  • When you click on the Configuration → Execution role → Role name you will redirected to the lambda permission section

🔑 6️⃣Create an API Gateway REST API (with Authorizer)

  1. Go to API Gateway → Create REST API.
  2. Resource: /users/me/image . (create resource in existing API path)
  3. Method: POST.
  4. Integration: Lambda function (created in the previous step).

5. Authorizer:

  • Add a Cognito User Pool Authorizer (or custom Lambda authorizer).
  • Secure the /users/me/image endpoint with this authorizer.
  • Method request settings -> edit for set authorizer
  • I already configured authorizer from cognito (playerAuth)
  • For Configuring cognito with authorizer — Hands-On (Will Udate Soon)
  • When we are uploading the image as binary (multi part file upload) we need to configure Binary media types
  • Go to API Gateway app settings → add the required file formats
  • After the POST method successfully configured deploy the API in created stage
