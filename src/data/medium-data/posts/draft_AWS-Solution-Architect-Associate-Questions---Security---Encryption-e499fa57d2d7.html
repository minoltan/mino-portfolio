A media company runs a photo-sharing web application that is accessed across three different countries. The application is deployed on several Amazon Elastic Compute Cloud (Amazon EC2) instances running behind an Application Load Balancer. With new government regulations, the company has been asked to block access from two countries and allow access only from the home country of the company.

Which configuration should be used to meet this changed requirement?

Overall explanation

Correct option:

AWS Web Application Firewall (AWS WAF) is a web application firewall service that lets you monitor web requests and protect your web applications from malicious requests. Use AWS WAF to block or allow requests based on conditions that you specify, such as the IP addresses. You can also use AWS WAF preconfigured protections to block common attacks like SQL injection or cross-site scripting.

Configure AWS Web Application Firewall (AWS WAF) on the Application Load Balancer in a Amazon Virtual Private Cloud (Amazon VPC)

You can use AWS WAF with your Application Load Balancer to allow or block requests based on the rules in a web access control list (web ACL). Geographic (Geo) Match Conditions in AWS WAF allows you to use AWS WAF to restrict application access based on the geographic location of your viewers. With geo match conditions you can choose the countries from which AWS WAF should allow access.

Geo match conditions are important for many customers. For example, legal and licensing requirements restrict some customers from delivering their applications outside certain countries. These customers can configure a whitelist that allows only viewers in those countries. Other customers need to prevent the downloading of their encrypted software by users in certain countries. These customers can configure a blacklist so that end-users from those countries are blocked from downloading their software.

Incorrect options:

Use Geo Restriction feature of Amazon CloudFront in a Amazon Virtual Private Cloud (Amazon VPC) — Geo Restriction feature of Amazon CloudFront helps in restricting traffic based on the user’s geographic location. But, CloudFront works from edge locations and doesn’t belong to a VPC. Hence, this option itself is incorrect and given only as a distractor.

Configure the security group on the Application Load Balancer

Configure the security group for the Amazon EC2 instances

Security Groups cannot restrict access based on the user’s geographic location.

1. Tell us about your app

  • App category: Click the dropdown and select the category that best fits (e.g., Social Media or Other).
  • App focus: Keep Both API and web selected (as shown in your screenshot), as photo-sharing apps usually use both.

2. Select resources to protect

  • Click Add AWS resources.
  • In the pop-up, select Application Load Balancer.
  • Choose your specific ALB from the list and click Add.

3. Choose initial protections

This is where the wizard differs from the manual setup.

  • Look for an option that says “Build your own pack” or “Custom protections”.
  • If the wizard forces you to choose a pre-configured pack, you can simply select a basic one (like “Core rule set”) and then click Next to reach the Rules section where you can add your specific Geo-Match rule.

4. Adding the Geo-Blocking Rule

Once you reach the Rules or Add Rules step:

  1. Click Add rules > Add my own rules and rule groups.
  2. Rule Builder:
  • Name: AllowHomeCountryOnly
  • Inspect: Originates from a country in
  • Country codes: Select your home country (e.g., Sri Lanka).
  • Action: Allow
  1. Default Action (Crucial):
  • Scroll to the bottom of the Rules page.
  • Set the Default action for the entire Web ACL to Block.
  • This logic ensures that if a user is from your home country, they are “Allowed” by the rule; otherwise, they fall through to the default “Block.”

5. Name and describe

  • Give your protection pack a name (e.g., PhotoApp-Regional-Restriction).
  • Review all settings and click Create protection pack (web ACL).

How to Apply Geo-Blocking to API Gateway

  1. Ensure your Web ACL is “Regional”: When you create the Web ACL in AWS WAF, make sure the Resource Type is set to Regional and the Region matches where your API Gateway is deployed.
  2. Go to API Gateway Console:
  • Select your REST API.
  • In the left-hand navigation pane, click on Stages.
  • Select the specific stage you want to protect (e.g., prod or v1).

3. Associate the Web ACL:

  • Under the Stage Details tab, look for the Web Application Firewall (WAF) section.
  • Click Edit.
  • In the Web ACL dropdown, select the Geo-blocking Web ACL you created earlier.
  • Click Save Changes.

To implement geo-blocking for an API Gateway REST API in CDK (TypeScript), you need to define two main components:

  1. A CfnWebACL: This contains your geo-match rule.
  2. A CfnWebACLAssociation: This links the WAF to your API Gateway Stage.

CDK Implementation (TypeScript)

Here is a clean snippet that sets up a “Whitelisted Country” logic.

import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as wafv2 from 'aws-cdk-lib/aws-wafv2';
import * as apigateway from 'aws-cdk-lib/aws-apigateway';
export class MyServerlessStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);
    // 1. Create the REST API
    const api = new apigateway.RestApi(this, 'MyPhotoApi', {
      restApiName: 'Photo Sharing Service',
    });
    api.root.addMethod('GET'); // Example method
    // 2. Define the Web ACL (Geo-Blocking)
    const geoBlockWebACL = new wafv2.CfnWebACL(this, 'GeoBlockWebACL', {
      defaultAction: { block: {} }, // Default: Block everyone
      scope: 'REGIONAL',
      visibilityConfig: {
        cloudWatchMetricsEnabled: true,
        metricName: 'GeoBlockMetric',
        sampledRequestsEnabled: true,
      },
      rules: [
        {
          name: 'AllowHomeCountry',
          priority: 1,
          action: { allow: {} }, // Action: Allow the specific country
          statement: {
            geoMatchStatement: {
              countryCodes: ['LK'], // Example: Sri Lanka
            },
          },
          visibilityConfig: {
            cloudWatchMetricsEnabled: true,
            metricName: 'AllowHomeCountryMetric',
            sampledRequestsEnabled: true,
          },
        },
      ],
    });
    // 3. Associate WAF with the API Gateway Stage
    // Note: Use the stage's ARN
    new wafv2.CfnWebACLAssociation(this, 'ApiAssociation', {
      resourceArn: api.deploymentStage.stageArn,
      webAclArn: geoBlockWebACL.attrArn,
    });
  }
}


CDK Implementation for ALB + WAF

Here is the TypeScript code to deploy an ALB and protect it with the same Geo-blocking logic:

import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import * as ec2 from 'aws-cdk-lib/aws-ec2';
import * as elbv2 from 'aws-cdk-lib/aws-elasticloadbalancingv2';
import * as wafv2 from 'aws-cdk-lib/aws-wafv2';
export class AlbWafStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);
    // 1. Setup VPC and ALB
    const vpc = new ec2.Vpc(this, 'MyVpc', { maxAzs: 2 });
    const alb = new elbv2.ApplicationLoadBalancer(this, 'MyALB', {
      vpc,
      internetFacing: true,
    });
    // Add a listener and a dummy target (e.g., for your photo app)
    const listener = alb.addListener('Listener', { port: 80 });
    listener.addAction('DefaultAction', {
      action: elbv2.ListenerAction.fixedResponse(200, {
        contentType: 'text/plain',
        messageBody: 'Welcome to the Home Country Photo App!',
      }),
    });
    // 2. Define the Web ACL
    const geoBlockWebACL = new wafv2.CfnWebACL(this, 'AlbGeoBlockWebACL', {
      defaultAction: { block: {} }, // Blocks all countries not explicitly allowed
      scope: 'REGIONAL',
      visibilityConfig: {
        cloudWatchMetricsEnabled: true,
        metricName: 'AlbGeoBlockMetric',
        sampledRequestsEnabled: true,
      },
      rules: [
        {
          name: 'AllowHomeCountryOnly',
          priority: 1,
          action: { allow: {} },
          statement: {
            geoMatchStatement: {
              countryCodes: ['LK'], // Your home country code
            },
          },
          visibilityConfig: {
            cloudWatchMetricsEnabled: true,
            metricName: 'AllowHomeCountry',
            sampledRequestsEnabled: true,
          },
        },
      ],
    });
    // 3. Associate WAF with the ALB
    // Use the Load Balancer ARN for the association
    new wafv2.CfnWebACLAssociation(this, 'AlbWafAssociation', {
      resourceArn: alb.loadBalancerArn,
      webAclArn: geoBlockWebACL.attrArn,
    });
    // Output the ALB DNS for testing
    new cdk.CfnOutput(this, 'ALBDNS', { value: alb.loadBalancerDnsName });
  }
}
