Welcome to this hands-on lab where we’ll set up Amazon CloudFront to deliver content stored in an Amazon S3 bucket. We’ll leverage Origin Access Control (OAC) to ensure our S3 content remains private and is only accessible through CloudFront, a crucial security best practice.

Prerequisites

Before we begin, ensure you have:

  • An AWS Account with sufficient permissions to create S3 buckets and CloudFront distributions.
  • Three sample files to upload to your S3 bucket. For this lab, we’ll use:

1. index.html (a simple HTML file with references to images)
2. coffee.jpeg
3. beach.jpeg

<!DOCTYPE html>

Step 1: Create an S3 Bucket and Upload Files

First, we need a place to store our website’s static files. We’ll create a private S3 bucket.

  1. Navigate to the S3 Console:
  • Log in to your AWS Management Console.
  • Search for “S3” in the search bar and select S3.

2. Create a New Bucket:

  • Click on Create bucket.
  • Bucket name: Enter a unique name for your bucket (e.g., demo-cloudfront-s3-v1).
  • AWS Region: Choose a region close to you or your target audience.
  • Leave Object Ownership as “ACLs disabled (recommended)”.
  • Ensure Block Public Access settings for this bucket remains enabled (all checkboxes checked). This is crucial for security.
  • Leave all other settings as default.
  • Click Create bucket.

3. Upload Files to Your Bucket:

  • Once your bucket is created, click on its name to enter it.
  • Click the Upload button.
  • Drag and drop your index.html, coffee.jpeg, and beach.jpeg files into the upload area, or click "Add files" and select them.
  • Leave all other settings as default.
  • Click Upload.

Verification:

  • After uploading, you should see your three files listed in the bucket.
  • Try opening index.html using its "Object URL" from the S3 console. You should receive an "Access Denied" error. This confirms your bucket and objects are private, which is exactly what we want for security.

Step 2: Create a CloudFront Distribution

Now, let’s create our CloudFront distribution and link it to our S3 bucket.

  1. Navigate to the CloudFront Console:
  • Search for “CloudFront” in the AWS Management Console and select CloudFront.
  • Notice that CloudFront is a global service, so there’s no region selection.

2. Create a New Distribution:

  • Click on Create distribution (demo-origin-test-v1).

3. Configure Origin:

  • Under Origin domain, select your S3 bucket from the dropdown list (e.g., demo-cloudfront-yourname-v4.s3.amazonaws.com). CloudFront will automatically populate this from your S3 buckets.
  • For Origin access, select Origin access control settings (recommended). This is the modern and secure way to grant CloudFront access to your private S3 bucket.
  • Click Create control setting.
  • A dialog box will appear. You can accept the default Name for the OAC or provide a descriptive one. Leave Signing behavior as “Sign requests (recommended)”.
  • Click Create.
  • You will see a prompt to “Update bucket policy”. Do not close this yet! We will use the generated policy.

4. Configure Default Cache Behavior:

  • Scroll down to Default cache behavior.
  • Leave Viewer protocol policy as “Redirect HTTP to HTTPS” or “HTTPS only” (recommended for production).
  • For Allowed HTTP methods, leave “GET, HEAD” (sufficient for serving static content).
  • Leave other cache behavior settings as default for this lab.

5. Configure Distribution Settings:

  • Scroll down to Settings.
  • For Price class, you can choose “Use all Edge Locations (best performance)” for a global audience, or a more restricted option for cost optimization. For this lab, “Use all Edge Locations” is fine.
  • For Default root object, enter index.html. This tells CloudFront which file to serve when a user accesses your distribution's root URL.
  • For Web Application Firewall (WAF), choose “Do not enable security protections” for this demo. (In a production environment, WAF is highly recommended).
  • Leave all other settings as default.

Step 3: Update S3 Bucket Policy

Now, we need to update our S3 bucket’s policy to allow the newly created CloudFront distribution to access its content.

  1. Copy the Generated Policy:
  • After clicking “Create distribution”, you should see a blue banner at the top of the CloudFront console with a message like “The S3 bucket policy needs to be updated…”.
  • Click Copy policy from this banner.
  • Then, click Go to S3 bucket permissions. This will take you directly to your S3 bucket’s permissions page.

Alternative (if you closed the banner):

  • Go back to the CloudFront console, click on your newly created distribution.
  • Go to the Origins tab.
  • Select your S3 origin and click Edit.
  • Scroll down, and you will find the Bucket policy section with a “Copy policy” button.

2. Paste the Policy into S3:

  • In the S3 bucket’s Permissions tab, scroll down to Bucket policy.
  • Click Edit.
  • Paste the copied policy into the policy editor.
  • Review the policy — it should grant s3:GetObject permission to the CloudFront service principal, with a condition ensuring it's for your specific CloudFront distribution's ARN. (For latest updated the permission are automatically get updated)
  • Click Save changes.

Verification:

  • Your S3 bucket policy now allows your CloudFront distribution to fetch objects, but not direct public access. This is the desired secure configuration.

Step 4: Test Your CloudFront Distribution

CloudFront distributions take some time to deploy (typically 5–15 minutes). You’ll see the status as “Deploying” in the CloudFront console.

  1. Wait for Deployment:
  • In the CloudFront console, go to Distributions.
  • Wait until the Status of your new distribution changes from “Deploying” to “Enabled”.

2. Access Your Content:

  • Once “Enabled”, copy the Distribution domain name (e.g., d123example.cloudfront.net).
  • Open a new web browser tab and paste the domain name.
  • You should now see your index.html page, with "I love Coffee!", "Hello World", and both images (coffee.jpeg and beach.jpeg) displayed correctly!

Try It:

  • Navigate directly to your image files using the CloudFront domain name, e.g., https://your-cloudfront-domain.net/coffee.jpeg and https://your-cloudfront-domain.net/beach.jpeg. They should load.
  • Try refreshing your page multiple times. Notice how quickly the content loads, especially after the first load. This is CloudFront serving content from its Edge cache!

Clean Up (Important!)

To avoid incurring unexpected costs, make sure to clean up the resources you’ve created.

  1. Delete CloudFront Distribution:
  • Go to the CloudFront console.
  • Select your distribution.
  • Click Disable (it might take a few minutes for the status to change to “Disabled”).
  • Once disabled, select the distribution again and click Delete. Confirm the deletion.

2. Delete S3 Bucket:

  • Go to the S3 console.
  • Select your bucket (demo-cloudfront-yourname-v4).
  • Click Empty and follow the prompts to confirm emptying the bucket.
  • Once empty, select the bucket again and click Delete. Follow the prompts to confirm deletion (you’ll need to type the bucket name).
