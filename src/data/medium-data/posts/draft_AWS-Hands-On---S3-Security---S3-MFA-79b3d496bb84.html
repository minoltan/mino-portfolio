This is a hands-on tutorial accompanying the blog post:

“Pass the AWS Certified Solutions Architect Associate Certification SAA-C03-(Episode 12: S3 Security)”

🔹 Focus: Practical implementation of Amazon S3.
🔹 For theoretical concepts and exam-style questions, please refer to the main blog post (link to parent blog).

Here’s a step-by-step guide to demonstrating S3 MFA Delete, based on the provided content.

Important Security Note: I used the AWS Root Account for this demonstration. It is highly recommended to avoid using your Root Account for daily operations. For enabling MFA Delete on a production bucket, it’s best practice to use an IAM user with the necessary permissions and MFA enabled for that user. This guide follows the provided content’s approach for demonstration purposes only.

Objective: Understand how to enable and test S3 MFA Delete for a bucket, which requires an MFA device and the AWS CLI.

Prerequisites:

✔ An AWS account.
✔ An MFA (Multi-Factor Authentication) device already set up for your AWS Root Account. (You can find this under IAM > Security credentials > Multi-factor authentication (MFA)). Make note of its ARN.
✔ AWS CLI installed and configured.
✔ A sample file (e.g., coffee.jpg) to upload.

Step 1: Create an S3 Bucket with Versioning Enabled

  1. Navigate to S3: Go to the Amazon S3 console.
  2. Create Bucket: Click “Create bucket”.
  3. Configure Bucket Details:
  • Bucket name: demo-mino-mfa-delete-2025 (or a unique name of your choice).
  • AWS Region: Choose a region, e.g., ap-southeast-1 (Singapore).
  • Bucket Versioning: Scroll down and “Enable” bucket versioning.
  • Leave other settings as default.

4. Create Bucket: Click “Create bucket”.

Step 2: Observe MFA Delete Status in Console (Initially Disabled)

  1. Navigate to the New Bucket: Go to your newly created bucket (demo-mino-mfa-delete-2025).
  2. Go to “Properties” Tab: Click on the “Properties” tab.
  3. Check Bucket Versioning: Scroll down to “Bucket Versioning” and click “Edit”.
  • Observation: You will see that “Multi-Factor authentication (MFA) delete” is currently Disabled, and there is no option to enable it directly in the console UI.

Step 3: (Temporary & Not Recommended for Production) Configure AWS CLI with Root Account Credentials

WARNING: Using Root Account access keys is a security risk. Delete these keys immediately after this demonstration.
  1. Generate Root Account Access Keys:
  • In the AWS console, click on your Root Account name (top right) and select “Security credentials”.
  • Under “Access keys (access key ID and secret access key)”, click “Create access key”.
  • Follow the prompts to generate a new access key ID and secret access key. Download the .csv file and copy the secret access key immediately, as you won't see it again.

2. Configure AWS CLI Profile:

  • Open your terminal/command prompt.
  • Run the command: aws configure --profile root-MFA-delete-demo (you can choose any profile name).
  • AWS Access Key ID: Paste the Access Key ID you just generated.
  • AWS Secret Access Key: Paste the Secret Access Key you just generated.
  • Default region name: Enter the region where your bucket is located (e.g., ap-southeast-1).
  • Default output format: Press Enter (default is fine).

3. Test CLI Configuration:

  • Run the command: aws s3 ls --profile root-MFA-delete-demo
  • Observation: You should see a list of your S3 buckets, confirming the CLI profile is set up correctly.

Step 4: Enable MFA Delete using AWS CLI

  1. Get Your MFA Device ARN:
  • Go back to the AWS console > Root Account name > “Security credentials”.
  • Under “Multi-factor authentication (MFA)”, copy the ARN of your virtual MFA device.

2. Construct the CLI Command: Use the following command. Replace YOUR_BUCKET_NAME with your actual bucket name and YOUR_MFA_DEVICE_ARN with the ARN you just copied.

aws s3api put-bucket-versioning \
--bucket YOUR_BUCKET_NAME \
--versioning-configuration '{"Status": "Enabled", "MfaDelete": "Enabled"}' \
--mfa "YOUR_MFA_DEVICE_ARN 123456" \
--profile root-MFA-delete-demo

3. Get Current MFA Code: Open your MFA application (e.g., Google Authenticator) and get the current 6-digit MFA code for your AWS account.

4. Execute the Command:

  • Paste the command into your terminal.
  • Crucially, replace 123456 with your current 6-digit MFA code.
  • Press Enter.
  • Observation: If successful, you won’t see any output. If it fails, you might get an “Invalid MFA code” error. Wait for a new code and try again.

Step 5: Verify MFA Delete is Enabled in the Console

  1. Navigate to Bucket Properties: Go back to your demo-stephane-mfa-delete-2020 S3 bucket in the AWS console.
  2. Go to “Properties” Tab: Click on the “Properties” tab.
  3. Check Bucket Versioning: Scroll down to “Bucket Versioning” and click “Edit”.
  • Observation: It should now explicitly state: “Multi-Factor authentication (MFA) delete is Enabled.”

Step 6: Test MFA Delete Protection

  1. Upload an Object:
  • Go to the “Objects” tab of your bucket.
  • Click “Upload”, then “Add files”, and select coffee.jpg.
  • Click “Upload”.

2. Attempt to Delete an Object (Creates Delete Marker — No MFA Prompt):

  • Select the coffee.jpg object.
  • Click “Delete”.
  • Type delete (not permanently delete) and confirm.
  • Observation: This will successfully create a “Delete marker” because versioning is enabled, and creating a delete marker doesn’t require MFA Delete.

3. Attempt to Permanently Delete a Specific Version (MFA Delete Blocks):

  • Go to the “Objects” tab.
  • Toggle “Show versions” (if not already enabled).
  • Select the actual object version of coffee.jpg (the one before the delete marker, with a non-null version ID).
  • Click “Delete”.
  • Observation: You will see a message like: “You cannot delete object because Multi-Factor authentication (MFA) delete is enabled for this bucket.” This confirms MFA Delete is working!

Step 7: Disable MFA Delete using AWS CLI (for Cleanup)

  1. Construct the CLI Command: Use the following command. Replace YOUR_BUCKET_NAME and YOUR_MFA_DEVICE_ARN as before.
aws s3api put-bucket-versioning \
--bucket YOUR_BUCKET_NAME \
--versioning-configuration '{"Status": "Enabled", "MfaDelete": "Disabled"}' \
--mfa "YOUR_MFA_DEVICE_ARN 123456" \
--profile root-MFA-delete-demo

2. Get Current MFA Code: Open your MFA application and get the current 6-digit MFA code.

3. Execute the Command:

  • Paste the command into your terminal.
  • Replace 123456 with your current 6-digit MFA code.
  • Press Enter.

Step 8: Verify MFA Delete is Disabled and Perform Permanent Delete

  1. Verify MFA Delete Status in Console:
  • Go back to your demo-stephane-mfa-delete-2020 S3 bucket in the AWS console.
  • Go to “Properties” > “Bucket Versioning” > “Edit”.
  • Observation: “Multi-Factor authentication (MFA) delete” should now be Disabled.

2. Permanently Delete a Specific Version (Now Allowed):

  • Go to the “Objects” tab.
  • Ensure “Show versions” is enabled.
  • Select the coffee.jpg delete marker (or any specific object version you want to permanently delete).
  • Click “Delete”.
  • Type permanently delete in the text box and click "Delete objects".
  • Observation: The deletion should now succeed.

Step 9: Clean Up (Crucial Security Step)

  1. Delete Your Root Account Access Keys:
  • In the AWS console, click on your Root Account name > “Security credentials”.
  • Under “Access keys (access key ID and secret access key)”, for the key you just created, click “Deactivate”, then “Delete”.
  • Confirm the deletion by typing the Access Key ID.
  • This is essential to secure your Root Account.

2. Delete the S3 Bucket: Delete the demo-stephane-mfa-delete-2020 bucket to avoid incurring storage costs.

Conclusion:

You have successfully demonstrated S3 MFA Delete. You’ve learned that:

  • MFA Delete can only be enabled/disabled via the AWS CLI or SDK.
  • It requires an MFA device associated with the account performing the operation.
  • It protects against accidental or malicious permanent deletion of object versions (including delete markers).
  • Creating a delete marker (soft delete) does not require MFA, but permanently deleting the underlying object version or the delete marker itself does (when MFA Delete is enabled).
