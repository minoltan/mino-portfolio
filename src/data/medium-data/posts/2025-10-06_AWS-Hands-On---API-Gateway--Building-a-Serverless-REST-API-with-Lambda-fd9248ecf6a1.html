Introduction

Hey cloud enthusiasts! Ready to get hands-on with Amazon API Gateway and AWS Lambda to build a serverless REST API? In this step-by-step guide, we’ll walk through creating a REST API in API Gateway, integrating it with Lambda functions, and deploying it for public access. This tutorial is perfect for AWS certification prep or anyone looking to master serverless APIs. Let’s dive into the AWS console and build a fully functional API from scratch!

Prerequisites

  • An AWS account with access to API Gateway and Lambda.
  • Basic familiarity with AWS console navigation.
  • Optional: A code editor to prepare Lambda code (we’ll provide a simple example).

Step 1: Setting Up the API Gateway

  1. Log into the AWS Management Console and navigate to the API Gateway service.

2. In the API Gateway console, you’ll see options for different API types:

  • HTTP API: Lightweight, cost-effective APIs.
  • WebSocket API: For real-time, bidirectional communication.
  • REST API: Feature-rich APIs (public or private).
  • For this tutorial, select REST API.

3. Click Build under REST API.

  • Note: AWS is transitioning to a new console experience, so ensure you’re using the updated interface (it will soon be the default).

4. Create a New API:

  • Choose New API.
  • Enter a name, e.g., MyFirstAPI.
  • Optionally, you can import an API from an OpenAPI definition, clone an existing API, or use an example API. We’ll start fresh.

5. Choose an Endpoint Type:

  • Regional: Deployed in one region, ideal for region-specific traffic.
  • Edge-Optimized: Uses CloudFront edge locations for global low-latency access.
  • Private: Accessible only within a VPC.
  • Select Regional for simplicity.

6. Click Create API.

Step 2: Creating a GET Method

  1. In the API Gateway console for MyFirstAPI, click Create Method under the root resource (/).

2. Select GET from the dropdown (other HTTP verbs like POST, PUT, DELETE are available).

3. Choose Integration Type:

  • Options include Lambda Function, HTTP, Mock, AWS Service, or VPC Link.
  • Select Lambda Function for this tutorial.

4. Lambda Proxy Integration:

  • Check Use Lambda Proxy integration to pass the full request to Lambda and receive the full response back.
  • Note: API Gateway has a 29-second timeout (non-configurable), regardless of Lambda’s timeout (up to 15 minutes).

5. You’ll need a Lambda function to integrate. Let’s create one next.

Step 3: Creating a Lambda Function

  1. Navigate to the Lambda service in the AWS console.
  2. Click Create Function:
  • Name: api-gateway-route-gets.
  • Runtime: Python 3.11 (other Python versions like Python 3.x should work).
  • Click Create Function.

3. Add Code:

  • Replace the default code with the following simple Python script:
import json

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'headers': {'Content-Type': 'application/json'},
        'body': json.dumps('Hello from Lambda!')
    }
  • This code returns a JSON response with a status code of 200 and a body of “Hello from Lambda!”.

4. Click Deploy to save and deploy the function.

5. Test the Function:

  • Create a test event (e.g., named DemoTest) with an empty JSON object ({}).
  • Click Test. You should see:
Response: {“statusCode”: 200, “headers”: {“Content-Type”: “application/json”}, “body”: “\”Hello from Lambda!\””}

Step 4: Integrating Lambda with API Gateway

  1. Return to the API Gateway console for MyFirstAPI.
  2. In the GET method setup:
  • Copy the ARN of the api-gateway-route-gets Lambda function from the Lambda console.
  • Paste the ARN into the Lambda Function field.
  • Ensure Use Lambda Proxy Integration is checked.

3. Click Save. API Gateway automatically creates an IAM policy allowing it to invoke the Lambda function.

4. Verify Permissions:

  • In the Lambda console, go to the api-gateway-route-gets function.
  • Under Configuration > Permissions, check the Resource-based policy.
  • Confirm that API Gateway has lambda:InvokeFunction permission for this function.

Step 5: Testing the GET Method

  1. In the API Gateway console, go to the GET method under the root resource (/).
  2. Click Test (bottom right).
  3. Leave query strings and headers empty for now, then click Test.
  4. Expected Output:
  • Status: 200
  • Response Body: “Hello from Lambda!”
  • Headers: Content-Type: application/json

5. Check the Execution Log in the test results to debug the request flow. This is useful for troubleshooting.

Step 6: Debugging with CloudWatch Logs

  1. Update the Lambda function to log the incoming event:
  • Modify the code to include:
import json

def lambda_handler(event, context):
    print(event)  # Log the event
    return {
        'statusCode': 200,
        'headers': {'Content-Type': 'application/json'},
        'body': json.dumps('Hello from Lambda!')
    }

2. Click Deploy.

3. Run another test in API Gateway.

4. Go to the Lambda console > Monitoring > View logs in CloudWatch.

5. Find the latest log stream and inspect the printed event. You’ll see details like:

  • resource: /
  • path: /
  • httpMethod: GET
  • headers, queryStringParameters, etc.

6. This event data can be used in your Lambda function to customize responses.

Step 7: Adding a New Resource and Method

  1. In API Gateway, click Create Resource:
  • Resource Name: houses
  • Path: /houses
  • Click Create Resource.

2. Under /houses, click Create Method and select GET.

3. Set Up Integration:

  • Choose Lambda Function.
  • Enable Lambda Proxy Integration.
  • Create a new Lambda function: ( Name: api-gateway-houses-gets, Runtime: Python 3.11 )
  • Code:
import json

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'headers': {'Content-Type': 'application/json'},
        'body': json.dumps('Hello from my pretty house!')
    }
  • Deploy the function.
  • Copy the new function’s ARN and paste it into the API Gateway GET method setup.

4. Click Save.

5. Test the /houses GET method:

  • Expected Output: Status 200, Body: “Hello from my pretty house!”

Step 8: Deploying the API

  1. In API Gateway, click Deploy API.
  2. Create a Stage:
  • Stage Name: dev
  • Click Deploy.

3. After deployment, copy the Invoke URL (e.g., https://<api-id>.execute-api.<region>.amazonaws.com/dev).

4. Test in a Browser:

  • Paste the Invoke URL in a web browser (e.g., https://<api-id>.execute-api.<region>.amazonaws.com/dev).
Result: {“message”: “Hello from Lambda!”}
  • Append /houses (e.g., https://<api-id>.execute-api.<region>.amazonaws.com/dev/houses).
Result: {“message”: “Hello from my pretty house!”}
  • Try an invalid path (e.g., /wrong):
Result: {“message”: “Missing Authentication Token”}

Step 9: Understanding the Setup

You’ve now created a serverless REST API with:

  • Two resources: / and /houses.
  • Two Lambda functions: api-gateway-route-gets and api-gateway-houses-gets.
  • A deployed API accessible via a public URL in the dev stage.

API Gateway handles:

  • Routing requests to the correct Lambda function.
  • Enforcing the 29-second timeout.
  • Automatic IAM permissions for Lambda invocation.
  • Logging for debugging (via CloudWatch).
