The Problem — Sarah’s Farm

Sarah owns a 100-acre farm in Iowa, divided into a neat 10×10 grid of plots. Each plot is 1 acre, and she grows different crops: corn, soybeans, wheat, and vegetables.

Every season, Sarah tracks the profit from each individual plot in a spreadsheet that looks like this:

Each number represents the profit (in thousands) from that specific acre.

The Business Questions Sarah Faces Daily:

  1. Insurance Adjuster “Sarah, the storm damaged your northeast section. What was the profit from plots in rows 0–2, columns 2–4?”
  2. Land Lease Decision “A neighbor wants to lease your western plots (rows 1–3, columns 0–1). What profit would you lose?”
  3. Crop Rotation Planning “Should I rotate crops in the southern quadrant? Let me calculate current profits from rows 3–4, columns 0–4.”
  4. Investment Analysis “Installing a new irrigation system costs $50,000 for the central region. What’s the current profit from rows 1–3, columns 1–3 to see if it’s worth it?”

Tom’s First Attempt (Brute Force)

Sarah’s nephew, Tom (who just learned programming), suggests: “Aunt Sarah, let me write you a program!”

“It’s simple! Just loop through all the plots in the rectangle and add them up!”

public class FarmProfitCalculator {
    private int[][] farmPlots;
    
  
    public FarmProfitCalculator(int[][] plots) {
        this.farmPlots = plots;
    }
    
   
    public int calculateRegionProfit(int row1, int col1, int row2, int col2) {
        int totalProfit = 0;
        
        // Loop through every row in the region
        for (int row = row1; row <= row2; row++) {
            // Loop through every column in that row
            for (int col = col1; col <= col2; col++) {
                totalProfit += farmPlots[row][col];
            }
        }
        
        return totalProfit;
    }
}

Complexity:

  • Time: O(M×N)
  • Space: O(1)

The Magic Formula

To get the sum of ANY rectangle, we use this formula:

🧠 Formula used

Each cell in the prefix sum array is computed using:

P(i,j)=A(i,j)+P(i−1,j)+P(i,j−1)−P(i−1,j−1)

Where:

  • A(i,j) = original value
  • P(i,j) = prefix sum at that cell

📊 Let’s break down Row 1

Original row 1:

PrefixSum = Current + TopArea + LeftArea — DoubleCounted
     0   1   2   3   4
0 [  3   2   1   4   5 ]
1 [  5   6   7   8   1 ]
2 [  2   3   4   5   6 ]
3 [  1   2   3   4   5 ]

At each cell:

➕ Add current value
➕ Add top area
➕ Add left area
➖ Remove double-counted overlap

👉 P(1,0)
= A(1,0) + P(0,0)
= 5 + 3 = 8

👉 P(1,1)
= A(1,1) + P(0,1) + P(1,0) - P(0,0)
= 6 + 5 + 8 - 3 = 16

👉 P(1,2)
= A(1,2) + P(0,2) + P(1,1) - P(0,1)
= 7 + 6 + 16 - 5 = 24

👉 P(1,3)
= A(1,3) + P(0,3) + P(1,2) - P(0,2)
= 8 + 10 + 24 - 6 = 36

👉 P(1,4)
= A(1,4) + P(0,4) + P(1,3) - P(0,3)
= 1 + 15 + 36 - 10 = 43
Sum Region = Total Area — Top Strip — Left Strip + The Correction

Build Prefix Sum Array

We create a (rows+1) × (cols+1) array = 5×6 array with extra row/column of zeros:

Why the extra row and column of zeros?

It makes the formula work for edge cases! When querying region starting at row 0 or column 0, we don’t get array index out of bounds errors.

Think of the extra row and column of zeros like a “safety fence” around Sarah’s farm.

When you calculate the prefix sum for a plot, the formula asks you to look at the plot to the left and the plot above.

The Problem Without Zeros

Imagine you are at the very first plot (Row 0, Col 0).

Formula: P(0,0) = A(0,0) + P(-1,0) + P(0,-1) — P(-1,-1)

The Crash: Computers don’t know what Row -1 or Column -1 is. The program will crash with an ArrayIndexOutOfBounds error because you are trying to look "off the map."

The Solution: The “Zero Buffer”

class FarmProfitCalculator {
    private int[][] prefixSum;
    
    public FarmProfitCalculator(int[][] matrix) {

        int rows = matrix.length;
        int cols = matrix[0].length;


        if (matrix == null || rows == 0 || cols == 0) {
            prefixSum = new int[0][0];
            return;
        }
        
        int rows = matrix.length;
        int cols = matrix[0].length;
        prefixSum = new int[rows + 1][cols + 1];
        
        for (int i = 1; i <= rows; i++) {
            for (int j = 1; j <= cols; j++) {
                prefixSum[i][j] = matrix[i-1][j-1] 
                                + prefixSum[i-1][j] 
                                + prefixSum[i][j-1] 
                                - prefixSum[i-1][j-1];
            }
        }
    }
    
    public int sumRegion(int row1, int col1, int row2, int col2) {
        row1++; col1++; row2++; col2++;
        return prefixSum[row2][col2] 
             - prefixSum[row1-1][col2] 
             - prefixSum[row2][col1-1] 
             + prefixSum[row1-1][col1-1];
    }
}

Complexity:

  • Time: O(1) per query, O(M×N) preprocessing
  • Space: O(M×N)
