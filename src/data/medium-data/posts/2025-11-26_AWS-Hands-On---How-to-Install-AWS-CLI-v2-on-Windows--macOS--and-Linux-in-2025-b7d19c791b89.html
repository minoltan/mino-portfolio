Introduction

Whether you’re building GenAI apps with Bedrock, managing S3 Vector Buckets, automating game catalog syncs, or just learning AWS — the AWS Command Line Interface (CLI) version 2 is your best friend. It’s faster, more secure, actively maintained, and has the exact same commands as v1 — so all tutorials (even old ones) still work perfectly. Here’s the fastest, most up-to-date way to install AWS CLI v2 on all three major operating systems as of 2025.

1. Install AWS CLI v2 on Windows

  1. Open your browser and go to: https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html
  2. Click the big Download AWS CLI button → Choose Windows (64-bit) → Direct link: https://awscli.amazonaws.com/AWSCLIV2.msi
  3. Double-click the downloaded .msi file.
  4. Click through the wizard:
  • Next → I accept the terms → Next
  • Keep default install location
  • Click Install → Yes (UAC prompt) → Finish

5. Open Command Prompt or PowerShell (no restart needed!)

6. Verify it worked:

aws --version
aws-cli/2.17.28 Python/3.11.8 Windows/10 exe/AMD64 prompt/off

Done! You now have the latest AWS CLI on Windows.

Pro tip: Add it to your PATH automatically (it does this by default). Use it in PowerShell, CMD, or Windows Terminal.

2. Install AWS CLI v2 on macOS

  1. Go to: https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html
  2. Under macOS → click Download AWS CLI → choose the .pkg for your architecture → Intel Macs: https://awscli.amazonaws.com/AWSCLIV2.pkg → Apple Silicon (M1/M2/M3): same link works universally now!
  3. Double-click the downloaded .pkg file.
  4. Click Continue → Continue → Agree → Install for all users of this computer (recommended) → Enter your password → Install
  5. Wait 20–30 seconds → Close the installer (move to Trash if prompted)
  6. Open Terminal (or iTerm2, Warp, etc.)

7. Check the version:

aws --version
aws-cli/2.17.28 Python/3.11.8 darwin/arm64 prompt/off

You’re ready to run aws s3 ls, aws bedrock list-foundation-models, or anything else!

3. Install AWS CLI v2 on Linux

Works on Ubuntu, Amazon Linux, Debian, Fedora, Arch, etc.

Open your terminal and run these three commands one by one:

# 1. Download the installer
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"

# For Apple Silicon Linux (rare) or ARM64:
# curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip"

# 2. Unzip it
unzip awscliv2.zip

# 3. Run the installer (sudo required)
sudo ./aws/install

That’s it!

Verify:

aws --version
aws-cli/2.17.28 Python/3.11.8 Linux/5.15.0–106-generic exe/x86_64.ubuntu.22 prompt/off

Conclusion

In just a couple of minutes and a few clicks (or three simple commands), you now have the latest AWS CLI v2 installed and ready on Windows, macOS, or Linux. From this single tool you can control every AWS service — S3 buckets, EC2 instances, Lambda functions, IAM policies, CloudFormation stacks, and hundreds more — directly from your terminal or scripts. No matter which operating system you or your team use, the installation process is straightforward, officially supported, and automatically stays up-to-date when you re-run the installer. With the CLI successfully installed and aws — version returning the current version, you’re fully equipped to manage your entire AWS environment efficiently, reproducibly, and at any scale. Welcome to the command-line superpower — happy automating!
