Objective

We will set up two EC2 instances in different AWS regions, each serving a simple “Hello World” web page. Then, we will configure AWS Global Accelerator to direct global traffic to the nearest healthy instance, demonstrating its latency-reducing and failover capabilities.

Part 1: Prepare Application Endpoints (EC2 Instances)

We’ll create two EC2 instances, one in US East (N. Virginia) and one in Asia Pacific (Mumbai), acting as our application endpoints.

Step 1.1: Create EC2 Instance in US East (N. Virginia)

  1. Navigate to EC2 Console:
  • Log in to your AWS Management Console.
  • In the top-right corner, select the region US East (N. Virginia) — us-east-1.
  • Search for “EC2” and select EC2 to go to the dashboard.

2. Launch Instance:

  • Click Launch instance.
  • Name and tags: Give it a name like ga-us-east-instance.
  • Application and OS Images (Amazon Machine Image): Select Amazon Linux 2 AMI (HVM) (usually the first option).
  • Instance type: Choose t2.micro (Free tier eligible).
  • Key pair (login): Select “Proceed without a key pair (Not recommended)”. Confirm the acknowledgment checkbox. (We won’t be SSHing into this instance).

3. Configure Instance Details (Advanced Details):

  • Scroll down to Advanced details.
  • Under User data, paste the following script:
#!/bin/bash
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo "<h1>Hello World from US East One!</h1>" > /var/www/html/index.html
  • This script installs Apache, starts it, and creates a simple index.html page indicating the region.

4. Network Settings:

  • Click Edit next to “Network settings”.
  • Security group: Click Create security group.
  • Security group name: global-accelerator-demo
  • Description: Allow HTTP traffic for Global Accelerator demo
  • Inbound security group rules | Add rule: | Type: HTTP | Source type: Anywhere-IPv4 (0.0.0.0/0)
  • Leave other network settings as default.

5. Launch Instance:

  • Click Launch instance.
  • Wait for the instance to start.

Step 1.2: Create EC2 Instance in Asia Pacific (Mumbai)

  1. Change Region:
  • In the top-right corner of the AWS Console, change the region to Asia Pacific (Mumbai) — ap-south-1.
  • Go to the EC2 dashboard for this region.

2. Launch Instance:

  • Click Launch instance
  • Name and tags: Give it a name like ga-mumbai-instance.
  • Application and OS Images (Amazon Machine Image): Select Amazon Linux 2 AMI (HVM).
  • Instance type: Choose t2.micro.
  • Key pair (login): Select “Proceed without a key pair”. Confirm acknowledgment.

3. Configure Instance Details (Advanced Details):

  • Scroll down to Advanced details.
  • Under User data, paste the following script:
#!/bin/bash
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo "<h1>Hello World from AP South One!</h1>" > /var/www/html/index.html

4. Network Settings:

  • Click Edit next to “Network settings”.
  • Security group: Click Create security group.
  • Security group name: global-accelerator-demo (Yes, reuse the name, but remember this is a new security group scoped to this region).
  • Description: Allow HTTP traffic for Global Accelerator demo
  • Inbound security group rules | Add rule | Type: HTTP| Source type: Anywhere-IPv4 (0.0.0.0/0)
  • Leave other network settings as default.
  • Launch Instance | Click Launch instance | Wait for the instance to start.

Step 1.3: Verify EC2 Instances are Accessible

  1. Go to the EC2 Instances page for US East (N. Virginia).
  2. Select ga-us-east-instance. Copy its Public IPv4 address or Public IPv4 DNS.
  3. Paste it into a new web browser tab. You should see: Hello World from US East One!
  4. Repeat for the ga-mumbai-instance in AP South (Mumbai). You should see: Hello World from AP South One!

If both instances display their respective “Hello World” messages, your endpoints are ready.

Part 2: Create AWS Global Accelerator

Now we’ll configure Global Accelerator to route traffic to our EC2 instances.

  1. Navigate to Global Accelerator Console:
  • Search for “Global Accelerator” in the AWS Management Console and select Global Accelerator.
  • Notice: Regardless of your current region, the console will redirect you to a specific configuration region (e.g., US West (Oregon)) as Global Accelerator is a global service.

2. Create Accelerator:

  • Click Create accelerator.
  • Accelerator name: my-first-accelerator
  • Click Next.

3. Add Listener:

  • Listeners define the entry points for traffic into your accelerator.
  • Port: 80
  • Protocol: TCP (HTTP uses TCP)
  • Client affinity: None (for this lab, we don't need sticky sessions)
  • Click Next.

4. Add Endpoint Groups:

  • Endpoint groups logically group your application’s endpoints (e.g., by region).
Add endpoint group 1:
  • Region: Select US East (N. Virginia) — us-east-1.
  • Traffic dial: 100% (meaning this group can receive 100% of traffic, along with other groups if applicable).
  • Health checks | Protocol: HTTP | Port: 80 | Path: / (checks the root of your web server) | Interval: 10 seconds | Threshold: 3 (3 consecutive successful health checks to be considered healthy)
  • Endpoints | Click Add endpoint. | Endpoint type: EC2 instance | Endpoint: Select your ga-us-east-instance from the dropdown. | Weight: Leave as 128 (default).
Add endpoint group 2:
  • Click Add endpoint group.
  • Region: Select Asia Pacific (Mumbai) — ap-south-1.
  • Traffic dial: 100%
  • Health checks: (Same as above) | Protocol: HTTP | Port: 80 |Path: /| Interval: 10 seconds | Threshold: 3
  • Endpoints: | Click Add endpoint. | Endpoint type: EC2 instance | Endpoint: Select your ga-mumbai-instance from the dropdown. | Weight: Leave as 128.

5. Create Accelerator:

  • Click Create accelerator.

Part 3: Test Global Accelerator and Failover

Global Accelerator will now start deploying and performing health checks. This can take a few minutes for the status to show “Healthy”.

  1. Monitor Health Status:
  • In the Global Accelerator console, click on your my-first-accelerator.
  • Go to the Listeners tab, then click on the Listener ID.
  • You’ll see your two Endpoint Groups. Monitor their Health status. Initially, it might show “Unhealthy” or “Unknown”. Wait until both show “Healthy”.

2. Test Global Accelerator’s Anycast IPs:

  • Once both endpoint groups are healthy, copy one of the Static IP addresses or the DNS name displayed on your accelerator’s overview page.
  • Open a new web browser tab and paste the IP address or DNS name.
  • You should now see either Hello World from US East One! or Hello World from AP South One!.

Geographic Routing Test (Optional — Requires VPN/Proxy):

  • If you are testing from Europe or America, you will likely connect to the US East (N. Virginia) instance, as it’s typically closer via the AWS network.
  • To see the Mumbai instance response, you would need to use a VPN or proxy service to simulate being in a region geographically closer to Mumbai (e.g., Southeast Asia, India, Australia).
  • Connect your VPN to a location like Indonesia or Singapore.
  • Refresh the Global Accelerator URL. You should now see Hello World from AP South One! This demonstrates Global Accelerator's intelligent routing to the closest endpoint.

3. Test Automated Failover:

Simulate Failure in Mumbai:
  • Go to the EC2 Instances page for Asia Pacific (Mumbai).
  • Select your ga-mumbai-instance.
  • Under the Security tab, click on the global-accelerator-demo security group.
  • Under Inbound rules, click Edit inbound rules.
  • Remove the HTTP rule (Port 80, Source 0.0.0.0/0).
  • Click Save rules.
Monitor Global Accelerator Health:
  • Go back to the Global Accelerator console, view your listener and endpoint groups.
  • Within seconds to a minute, the Health status for the ap-south-1 endpoint group should change to "Unhealthy".
Observe Traffic Rerouting:
  • Go back to the web browser tab with your Global Accelerator URL.
  • Refresh the page.
  • You should now consistently receive the response: Hello World from US East One!, even if you are simulating a location closer to Mumbai. Global Accelerator automatically detected the unhealthy endpoint and rerouted all traffic to the healthy US East instance.

Part 4: Clean Up (Crucial to Avoid Costs!)

Follow these steps precisely to avoid ongoing charges.

  1. Terminate EC2 Instances:
  • Go to the EC2 Instances page for US East (N. Virginia).
  • Select ga-us-east-instance.
  • Click Instance state -> Terminate instance. Confirm termination.
  • Go to the EC2 Instances page for Asia Pacific (Mumbai).
  • Select ga-mumbai-instance.
  • Click Instance state -> Terminate instance. Confirm termination.

2. Delete Global Accelerator:

  • Go to the Global Accelerator console.
  • Select your my-first-accelerator.
  • Click Disable accelerator. This will take a few minutes. Wait until the status changes to “Disabled”.
  • Once disabled, select the accelerator again.
  • Click Delete accelerator. Type delete in the confirmation box and click Delete.
