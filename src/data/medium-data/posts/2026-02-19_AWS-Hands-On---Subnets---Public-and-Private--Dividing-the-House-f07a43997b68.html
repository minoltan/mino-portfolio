🌟 Introduction

Now that we have built the building (our VPC). It’s time to frame the rooms. In AWS, subnets allow you to slice your large network into smaller, manageable segments. Think of a public subnet as your front door. It's where you put things that need to interact with the world, like a web server or a load balancer. A private subnet is like your locked basement. It’s where you keep the valuables, like your database, away from direct public access. By spreading these subnets across multiple Availability Zones (AZs), we ensure that if one data center has an issue, our app stays alive in another.

Step 1: Setting Up the Front Porch (Public Subnets)

We’ll start by creating two small subnets for our public-facing resources. We use a smaller range (/24) here because we typically don't need thousands of IPs for load balancers.

  1. Navigate to Subnets and click Create subnet.
  2. VPC ID: Select your DemoVPC.
  3. Subnet 1:
  • Name: PublicSubnetA
  • Availability Zone: ap-southeasta-1a
  • CIDR Block: 10.0.0.0/24 (256 IP addresses)

4. Subnet 2:

  • Name: PublicSubnetB
  • Availability Zone: ap-southeasta-1b
  • CIDR Block: 10.0.1.0/24 (The next logical range)

Step 2: Building the Vault (Private Subnets)

Next, we’ll create two larger subnets for our application servers and databases. These need more room (/20) to allow for scaling.

  1. Subnet 3:
  • Name:PrivateSubnetA
  • Availability Zone:ap-southeasta-1a
  • CIDR Block: 10.0.16.0/20 (4,096 IP addresses)

2. Subnet 4:

  • Name:PrivateSubnetB
  • Availability Zone:ap-southeasta-1b
  • CIDR Block: 10.0.32.0/20 (Another 4,096 IPs)

The “Missing” 5 IPs: AWS Reservations

You might notice that even though a /24 mathematically has 256 addresses, AWS shows only 251 available. This isn't a glitch. In every subnet, AWS reserves the first four and the last one IP addresses for its own use

  • 10.0.0.0: Network address.
  • 10.0.0.1: VPC Router.
  • 10.0.0.2: DNS mapping.
  • 10.0.0.3: Future use.
  • 10.0.0.255: Network broadcast address (AWS doesn’t support broadcast, but reserves it anyway).

🧾 Conclusion

We now have a multi-AZ layout ready for a high-availability application. We’ve carved out space for the public to visit and a secure area for our data to live. However, right now, these are just names. A “Public” subnet isn’t actually public until we connect it to the internet through a gateway and update its routing instructions.
