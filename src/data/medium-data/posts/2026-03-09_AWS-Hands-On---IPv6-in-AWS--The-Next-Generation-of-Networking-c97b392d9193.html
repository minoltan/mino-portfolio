🌟 Introduction

While IPv4 is still the standard, the world is slowly running out of addresses. IPv6 was designed to solve this problem by offering an almost infinite number of IPs. In AWS, this is implemented through a “Dual-Stack” configuration, where resources can have both IPv4 and IPv6 addresses simultaneously.

Introduction

Enabling IPv6 in your VPC is a multi-step process that transitions your network into a Dual-Stack environment. Unlike IPv4, which often relies on private ranges and NAT for internet access, every IPv6 address in AWS is publicly routable by default. However, this doesn’t mean your instances are automatically exposed; you still use Security Groups and Route Tables to control traffic. By assigning an Amazon-provided /56 CIDR block to your VPC and subdividing it into /64 ranges for your subnets, you provide your instances with a modern, scalable networking layer that bypasses the limitations of IPv4 address exhaustion.

Step-by-Step Guide: Enabling IPv6

1. Add an IPv6 CIDR to the VPC

A VPC doesn’t have IPv6 by default; you must request a block.

  • Go to VPC > Your VPCs.
  • Select your VPC, click Actions, and choose Edit CIDRs.
  • Click Add new IPv6 CIDR.
  • Select Amazon-provided IPv6 CIDR block (AWS will assign you a /56 range).
  • Click Select CIDR and then Close.

2. Configure Subnet IPv6 Ranges

Once the VPC has a large block, you must carve out a piece for each subnet.

  • Go to Subnets and select your public subnet.
  • Click Actions > Edit IPv6 CIDRs.
  • Add a /64 CIDR (e.g., using 00 as the hex identifier).
  • Auto-assign Setting: While in Subnet settings, check Enable auto-assign IPv6 address so new instances get one automatically.

3. Update the Route Table

Your instances need a “map” to find the IPv6 internet.

  • Go to your Public Route Table > Routes > Edit routes.
  • Local Route: Notice that a new local route for your IPv6 range was added automatically. This allows internal VPC communication.
  • Public Route: Add a route for ::/0 (the IPv6 equivalent of 0.0.0.0/0).
  • Target: Select your Internet Gateway.

4. Assign an IPv6 Address to an Instance

If you have an existing instance, you can add an IPv6 address manually.

  • Go to EC2 > Instances.
  • Select your Bastion Host, click Networking > Manage IP addresses.
  • Under the interface, click Assign new IPv6 address (leave it on Auto-assign).
  • Click Save and Confirm.

5. Update Security Groups

Your IPv4 security rules do not cover IPv6 traffic.

  • Go to your Security Group > Inbound Rules > Edit.
  • Add an SSH rule.
  • Source: Select ::/0 (Anywhere IPv6).
  • Note: You can use sites like Test-IPv6.com to check if your current internet connection supports IPv6 so you can test this connection.

Important Differences to Remember

🧾 Conclusion

Adding IPv6 to your VPC ensures your infrastructure is future-proof and capable of communicating with the growing number of IPv6-only devices globally. One of the most important takeaways from this practice is that IPv4 remains mandatory — every instance in a dual-stack VPC still consumes one of your limited IPv4 addresses. While IPv6 provides a massive address space, you must still manage your IPv4 CIDR blocks carefully to avoid running out of capacity for your instances.
