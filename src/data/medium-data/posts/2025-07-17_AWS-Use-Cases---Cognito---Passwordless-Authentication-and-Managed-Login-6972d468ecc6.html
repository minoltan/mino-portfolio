AWS Use Cases | Cognito | Passwordless Authentication and Managed Login

Introduction

Amazon Cognito, launched over a decade ago, has matured into a core service for implementing Customer Identity and Access Management (CIAM) in web and mobile applications. Whether you’re managing user authentication, enabling secure sign-in experiences, or scaling identity workflows, Cognito provides the backbone for secure and scalable user management.

Recently, AWS introduced significant enhancements to Cognito — designed to make authentication flows more secure, developer-friendly, and password-free. These updates include:

  • A revamped developer console
  • Managed Login with visual customization
  • Passwordless authentication via passkeys, email, and SMS OTP
  • Flexible pricing tiers tailored to different business needs

Let’s explore how to set this up and the benefits these new features bring.

✅ Prerequisites

Before you begin:

  • An active AWS account
  • IAM permissions to create and manage Cognito resources

🔧 A New Developer Console Experience

The new Cognito developer console offers a streamlined setup that significantly reduces the time required to configure authentication. AWS now enables passwordless login using:

  • 📩 Email OTP
  • 📱 SMS OTP
  • 🔑 Passkeys (biometric, hardware keys, authenticator apps)

🧭 Step 1: Create a Cognito User Pool

  1. Log in to the AWS Console

2. Navigate to Amazon Cognito → Create user pool

3. Select your application type:

  • Traditional Web App
  • SPA (Single Page App) — selected for this guide
  • Mobile App
  • Machine to Machine

4. Define User Pool Name: Example: MyAppUserPool

👥 Step 2: Configure Sign-In Options

  1. Choose Sign-In Method
  • Enable email, phone number, or username (or any combo).
  • For passwordless: Enable email or phone number.

2. Select required attributes (e.g., email)

3. Add Return URL (optional)

4. Then, select Create user directory.

Once done, the user pool dashboard shows up with an overview and recommendations. It includes a visual preview of your login flows and lets you customize everything easily.

  • Click on Go to overview on the bottom button, you will see the overview of the created user pool.

On this page, you can customize your users’ sign-in and sign-up experience with the Managed Login feature. This is a good segue for me to provide you with a quick overview of the next new feature.

🎨 Introducing Managed Login

Managed Login is the new, modern take on Cognito’s classic Hosted UI. It provides:

  • Fully managed UI for sign-in, sign-up, MFA, and password reset
  • Built-in responsiveness for mobile and desktop
  • No-code Branding Designer for visual customization
  • Seamless updates and built-in security patches
  • Programmatic setup via CloudFormation

Domain Setup Options (Not Mandatory)

  • Use AWS-hosted subdomain (e.g., myapp.auth.us-east-1.amazoncognito.com)
  • Or bring your custom domain for brand consistency

Choose Managed Login over the Classic Hosted UI for a more modern, flexible, and responsive experience.

If you’re an existing Amazon Cognito user, you might be familiar with the classic Hosted UI feature. Managed Login is the improved version of Hosted UI, offering a new collection of web interfaces for sign-up and sign-in, built-in responsiveness for different screen sizes, multi-factor authentication, and password-reset activities in your user pool.

🔐 Passwordless Authentication is Here

With passwordless authentication, Cognito now supports:

  • Passkeys (device biometrics, secure keys)
  • Email OTP
  • SMS OTP

These options improve both user experience and security, reducing the reliance on traditional passwords.

🧩 Step 3: Set Up Passwordless Authentication

  1. Go to User Pool → Sign-in
  2. Edit → Options for choice-based sign-in
  3. Under Authentication Methods, enable:
  • Passkey Authentication
  • Email OTP
  • SMS OTP (to avoid extra configs I did not used SMS option)

4. Configure Message Settings (Optional)

By reducing the friction associated with traditional password-based sign-ins, this feature simplifies application access for your users while maintaining high security standards.

🧪 Test the passwordless login

  • To access sign-in / sign-up page, go to app client of a user pool and click on View login page button
  • At first need to Sign up using Create an account link
  • Provide the email address and preferred password (this password not mandatory on login)
  • Once you entered the valid email address you asked to enter the code that shared on your mail
  • The final step is to add passkey, you can attach your fingerprint as your preferred login.
  • When the user created the account successfully, you can validate on the user list.

🎉 You now have a fully working passwordless sign-in flow!

📊 More options on pricing tiers

Amazon Cognito has introduced new user pool feature tiers: Lite, Essentials, and Plus. These tiers are designed to cater to different customer needs and use cases with the Essentials tier being the default tier for new users pools created by customers.

To check your current tier, you can go to your application dashboard and select Feature plan. You can also select Settings from the navigation menu.

On this page, you’ll get detailed information for each tier and the option to downgrade or upgrade your plan.

Billing is based on Monthly Active Users (MAUs), and switching between tiers is supported from the dashboard.

Pros & Cons

Conclusion

With the latest upgrades, Amazon Cognito is more powerful and accessible than ever. Whether you’re building a startup MVP or scaling to millions of users, Cognito’s Managed Login, passwordless authentication, and flexible pricing make it an excellent choice for modern identity management.

The move to passwordless reduces friction and increases security — meeting users’ growing expectations for seamless sign-in experiences. Combined with intuitive tools like the Branding Designer and tiered pricing, Cognito makes secure authentication not only easy but scalable.

If you’re building a secure app in 2025, Cognito deserves a serious look.
