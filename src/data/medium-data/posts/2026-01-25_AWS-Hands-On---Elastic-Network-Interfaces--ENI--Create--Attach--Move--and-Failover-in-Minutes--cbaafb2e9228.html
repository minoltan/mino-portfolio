🌟 Introduction

When you launch an EC2 instance, it comes with a default network connection. But did you know you can customize how your instances talk to the network? This is where the Elastic Network Interface (ENI) comes into play. Think of an ENI as a virtual network card that you can plug in and pull out of your cloud servers. In this beginner-friendly hands-on guide, you’ll launch two instances, create your own ENI, attach it to give an instance a second private IP, move it between instances for a lightning-fast failover demo, and see what happens when you terminate everything. Takes less than 10 minutes and costs pennies (Free Tier friendly). Let’s jump in!

Step 1: Launch Two Test Instances

First, we need some hardware to plug our network cards into.

  • Launch two Amazon Linux 2 (t2.micro) instances.
  • Ensure they are in the same Availability Zone (ap-southeast-1b ).
  • Assign a default security group.
  • Once launched, check the Networking tab. You’ll see each instance already has one primary ENI attached automatically.

Step 2: Create Your Own ENI

  1. Now, let’s build our own custom network interface.
  • On the left sidebar -> under Network & Security -> click Network Interfaces.
  • Click Create Network Interface.
  • Description: Enter DemoENI.
  • Subnet: Choose the exact same subnet/AZ where your instances are located (ap-southeast-1b).
  • IPv4 Private IP: Select Auto-assign.
  • Security Groups: Select your active security group.
  • Click Create.

Step 3: Attach It to Instance-A

Your new ENI is currently available but not In-use.

  • Select your Demo ENI, click Actions, then Attach.
  • Choose your first EC2 instance.
  • Now, go back to your EC2 Instance dashboard. Under networking, you will see two network interfaces. Your instance now has its original primary IP and the new secondary private IP from the Demo ENI.

Step 4: The Cool Failover Demo — Move the ENI!

This is the “cool” part. Imagine Instance A crashes, but you want Instance B to immediately take over its private IP address.

  • Go back to Network Interfaces.
  • Select the Demo ENI, click Actions, and select Detach (use Force Detach if it's being stubborn).
  • Once the status returns to “Available,” click Actions -> Attach.
  • This time, select your second EC2 instance.
  • Result: The private IP address has effectively “moved” from the first server to the second. Any internal traffic pointing to that IP is now hitting the new server!

Step 6: Clean Up (Important!)

  • Terminate both of your EC2 instances.
  • Refresh your Network Interfaces list.
  • The Observation: The two ENIs that were created with the instances are gone. However, your manually created one Demo ENI is still there!
  • Manual Deletion: Since you created it manually, you must delete it manually. Select it and click Delete.

🧾 Conclusion

Now you’ve mastered one of AWS’s most powerful networking tricks. No reboots, no downtime, no complex scripts. ENIs are free (you only pay for IPs/data) and the secret behind rock-solid internal services, monitoring tools, and quick disaster recovery. Practice moving that Demo ENI a few more times, it’ll feel like magic every time! Next up: we’ll use ENIs with load balancers and VPC tricks. great job! 🚀
