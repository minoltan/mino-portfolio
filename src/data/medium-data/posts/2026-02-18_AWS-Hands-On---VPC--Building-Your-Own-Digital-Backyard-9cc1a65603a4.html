🌟 Introduction

Think of an AWS VPC (Virtual Private Cloud) as your own private data center in the cloud. In the real world, if you were building a physical office, the first thing you’d do is lease a building and decide on its internal layout. In AWS, creating a VPC is exactly that you are carving out a logically isolated section of the AWS network where you have complete control over your digital floor plan. While AWS offers a “VPC Wizard” that does the work for you, building it manually is the best way to understand how the plumbing of the cloud actually works.

Step 1: Naming and IP Planning (CIDR Blocks)

The first step in creating a VPC is defining its boundaries using a CIDR block. This determines how many IP addresses your network will have.

  1. Open the VPC Console and click Create VPC.

2. Choose VPC only (we’re doing this manually to learn!).

3. Name tag: Enter DemoVPC.

4. IPv4 CIDR block config: Enter 10.0.0.0/16.

  • The Math: Using /16 can get 65,536 IP addresses.
  • The Rule: AWS allows CIDR sizes between /16 (large) and /28 (small). You can't go bigger than /16 (like /15) because the network would be too massive to manage efficiently.

Step 2: Choosing Your Tenancy

Tenancy defines the physical hardware your resources will run on.

  • Default (Shared): Your instances run on the same physical hardware as other AWS customers (logically isolated). This is the standard, cost-effective choice.
  • Dedicated: Your instances run on hardware physically dedicated to only your account. This is significantly more expensive and usually only required for specific compliance or archaic software licensing rules.

For our demo, we’ll stick with Default.

Step 3: What Happens Behind the Scenes?

When you click Create VPC, AWS doesn’t just give you a name and an IP range. It automatically creates three essential “invisible” components for you:

  1. A Main Route Table: This acts as the internal traffic controller for your network.
  2. A Main Network ACL (NACL): This is the first layer of security (the “bouncer” at the subnet gate).
  3. A Default Security Group: The second layer of security (firewall for individual instances).

Step 4: Future-Proofing (Adding CIDRs)

Real-world businesses grow, and sometimes that initial /16 block isn't enough. AWS allows you to add up to five IPv4 CIDR blocks to a single VPC.

  • To do this, you can go to Actions > Edit CIDRs after the VPC is created.
  • You could add a secondary range like 10.1.0.0/16. This is handy for large-scale enterprise environments that need to expand their network without creating a whole new VPC.
  • You can have up to 5 CIDR blocks.

🧾 Conclusion

Congratulations! You’ve just initiated the foundation for your cloud infrastructure. By building your VPC manually, you’ve learned that a network is more than just a name. It’s a carefully defined space with its own IP range, hardware tenancy, and default security rules. Right now, your VPC is an isolated island. It can’t talk to the internet yet.
