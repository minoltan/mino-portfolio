🌟 Introduction

VPC Flow Logs provide a granular look at your network traffic without impacting instance performance. By capturing details like source/destination IPs, ports, and whether traffic was “Accepted” or “Rejected” (based on your Security Groups and NACLs), you gain the visibility needed to diagnose connectivity issues or spot malicious scanning activity. In this guide, we’ll set up two streams: one to CloudWatch Logs for immediate monitoring and one to Amazon S3 for deeper, high-volume analysis using Amazon Athena.

Step-by-Step Guide: Flow Logs and Athena Analysis

1. Create a Flow Log for Amazon S3

First, we’ll set up a long-term storage destination.

  • Create an S3 Bucket: e.g., demo-mino-vpc-flow-logs-v1.
  • Configure the Flow Log:
  • Navigate to VPC > Your VPCs > Select your VPC > Flow Logs tab.
  • Name: DemoS3FlowLog.
  • Filter: Select All (to see both allows and rejects).
  • Destination: Send to an Amazon S3 bucket.
  • S3 Bucket ARN: Paste your bucket’s ARN.
  • Aggregation Interval: 1 minute (for faster testing).

2. Create a Flow Log for CloudWatch Logs

This is ideal for seeing logs in the console almost instantly.

  • IAM Role: Create a role (e.g., flowlogsrole) with a trust policy allowing vpc-flow-logs.amazonaws.com to assume it, and attach CloudWatchLogsFullAccess.
  • CloudWatch Log Group: Create one named VPCFlowLogs.
  • Configure the Flow Log:
  • Create a second flow log in the VPC console.
  • Destination: CloudWatch Logs.
  • Destination Log Group: VPCFlowLogs.
  • IAM Role: Select your flowlogsrole.

3. Analyze Logs in CloudWatch

  • Go to CloudWatch > Log Groups > VPCFlowLogs.
  • Each Log Stream corresponds to a specific ENI (Elastic Network Interface).
  • Look for your Bastion Host’s ENI ID. You will likely see REJECT entries—these are often automated bots on the internet scanning for open ports.

4. Querying S3 Logs with Amazon Athena

Athena allows you to run SQL queries directly against your log files in S3.

  • Set up Athena: Specify an S3 bucket for query results in Athena’s settings (e.g., s3://athena-results-bucket/).
  • Create Table: Run a CREATE EXTERNAL TABLE statement. You can find the standard schema in the AWS Documentation.
CREATE EXTERNAL TABLE IF NOT EXISTS `vpc_flow_logs` (
  version int,
  account_id string,
  interface_id string,
  srcaddr string,
  dstaddr string,
  srcport int,
  dstport int,
  protocol bigint,
  packets bigint,
  bytes bigint,
  start bigint,
  `end` bigint,
  action string,
  log_status string,
  vpc_id string,
  subnet_id string,
  instance_id string,
  tcp_flags int,
  type string,
  pkt_srcaddr string,
  pkt_dstaddr string,
  region string,
  az_id string,
  sublocation_type string,
  sublocation_id string,
  pkt_src_aws_service string,
  pkt_dst_aws_service string,
  flow_direction string,
  traffic_path int
)
PARTITIONED BY (`date` date)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ' '
LOCATION 's3://amzn-s3-demo-bucket/prefix/AWSLogs/{account_id}/vpcflowlogs/{region_code}/'
TBLPROPERTIES ("skip.header.line.count"="1");
  • Add Partitions: Because flow logs are stored in a date-based folder structure, you must run an ALTER TABLE command to point Athena to the specific day's data:
ALTER TABLE vpc_flow_logs ADD PARTITION (date='2026-02-14')  LOCATION 's3://your-bucket/AWSLogs/your-id/vpcflowlogs/region/2026/02/14';
  • Run a Query: Find who is being blocked most often:
SELECT srcaddr, count(*) AS total_rejects 
FROM vpc_flow_logs 
WHERE action = 'REJECT' 
GROUP BY srcaddr 
ORDER BY total_rejects DESC 
LIMIT 10;

🧾 Conclusion

Setting up VPC Flow Logs transformed our “silent” network into a detailed stream of intelligence. We saw how CloudWatch Logs provides quick access to ENI-level data for troubleshooting, while Amazon S3 and Athena empower us to perform large-scale security audits and identify patterns across thousands of records using simple SQL. Remember to delete your flow logs and empty your S3 buckets after practicing to avoid ongoing storage costs. You now have the tools to see exactly what is happening inside your virtual network!
