Here’s a step-by-step guide to practicing S3 Replication, based on the provided content:

Objective: Understand and configure S3 Cross-Region Replication (CRR) and observe how objects and delete markers are replicated between buckets.

Prerequisites:

✔ An AWS account with access to Amazon S3.
✔ Two distinct S3 buckets.
✔ Sample image files (e.g., beach.jpg, coffee.jpg).

Step 1: Create the Origin S3 Bucket

  1. Create a New Bucket: In the Amazon S3 console, click “Create bucket”.
  2. Configure Bucket Details:
  • Bucket name: s3-mino-replication-v2 (or a unique name of your choice).
  • AWS Region: Choose a region, e.g., ap-southeast-1 (Asia - Singapore).

3. Enable Versioning: Scroll down to “Bucket Versioning” and select “Enable”. This is mandatory for replication.

4. Create Bucket: Click “Create bucket”.

Step 2: Create the Target S3 Bucket

  1. Create a New Bucket (New Tab Recommended): Open another S3 console tab or window and click “Create bucket”.
  2. Configure Bucket Details:
  • Bucket name: s3-mino-replication-replica-v2 (or a unique name).
  • AWS Region: Choose a different region for Cross-Region Replication, e.g., us-east-1 (US East - N. Virginia).
  1. Enable Versioning: Scroll down to “Bucket Versioning” and select “Enable”. This is also mandatory for replication.
  2. Create Bucket: Click “Create bucket”.

Step 3: Upload an Initial File to the Origin Bucket (Pre-Replication Setup)

  1. Navigate to the Origin Bucket: Go to your s3-mino-replication-v2 bucket.
  2. Upload a File: Click “Upload”, then “Add files”, and select beach.jpg (or any other image).
  3. Complete Upload: Click “Upload”.
Observation: This file will not be automatically replicated because the replication rule is not yet configured.

Step 4: Configure the Replication Rule on the Origin Bucket

  1. Go to “Management” Tab: In your s3-mino-replication-v2 (origin) bucket, click the "Management" tab.
  2. Create Replication Rule: Scroll down to “Replication rules” and click “Create replication rule”.
  3. Configure Rule Settings:
  • Replication rule name: DemoReplicationRule (or a descriptive name).
  • Status: Ensure “Enabled” is selected.
  • Source bucket: Leave as “Apply to all objects in the bucket”.
  • Destination: Choose a bucket in this account: Select this option. Bucket name: Copy and paste the exact name of your target bucket (s3-mino-replication-replica-v2). The destination region (us-east-1 in this example) should be automatically identified.
  • IAM role: Select “Create new role”. S3 will automatically create an IAM role with the necessary permissions for replication.
  • Encryption: (Optional, beyond this guide)
  • Replication metrics: (Optional)
  • Replication time control (RTC): (Optional, beyond this guide)

4. Save Rule: Click “Save changes”.

5. Replicate Existing Objects? When prompted “Do you want to replicate existing objects?”, select “No, do not replicate existing objects”.

Explanation: Replication rules only apply to new objects uploaded after the rule is created. Replicating existing objects requires an S3 Batch Operation, which is a separate process.

Step 5: Test Replication with a New Object

  1. Upload a New File to Origin: In your s3-mino-replication-v2 (origin) bucket, click "Upload", add coffee.jpg (or another file), and complete the upload.
Verify Version ID (Optional): In the origin bucket, enable “Show versions” and note the version ID of coffee.jpg.

2. Check Target Bucket: Go to your s3-mino-replication-replica-v2 (target) bucket and refresh the page.

Observation: After a short delay (a few seconds to a minute), coffee.jpg should appear in the target bucket.
Verify Version ID (Optional): Enable “Show versions” in the target bucket. The version ID of coffee.jpg in the target bucket should be the exact same as in the origin bucket.

3. Upload Another File to Origin: Upload beach.jpg again to your origin bucket. This creates a new version of beach.jpg in the origin bucket, which will then be replicated.

Observation: A new version of beach.jpg will appear in the target bucket shortly after the upload to the origin bucket.

Step 6: Enable and Test Delete Marker Replication

  1. Edit Replication Rule: In your s3-mino-replication-v2 (origin) bucket, go to the "Management" tab and select your DemoReplicationRule. Click "Edit".
  2. Enable Delete Marker Replication: Scroll down to the “Options” section and check the box for “Replicate delete markers”.
  3. Save Rule: Click “Save changes”.
  4. Delete an Object in Origin (Soft Delete): In your s3-mino-replication-v2 (origin) bucket, select coffee.jpg and click "Delete". Type delete (not permanently delete) and confirm.
Observation (Origin): When “Show versions” is enabled in the origin bucket, you will see a “Delete marker” for coffee.jpg. If "Show versions" is disabled, coffee.jpg appears to be gone.

5. Check Target Bucket: Go to your s3-mino-replication-replica-v2 (target) bucket and refresh.

Observation (Target): After a short delay, you will also see a “Delete marker” for coffee.jpg in the target bucket (if "Show versions" is enabled). If "Show versions" is disabled, coffee.jpg will also appear to be gone in the target bucket.

Step 7: Observe Permanent Delete Behavior (Not Replicated)

  1. Perform a Permanent Delete in Origin: In your s3-mino-replication-v2 (origin) bucket, ensure "Show versions" is enabled. Select an older specific version of beach.jpg (not the latest one or a delete marker) and click "Delete". Type permanently delete and confirm.
Explanation: Deleting a specific version ID is a permanent delete of that version.

2. Check Target Bucket: Go to your s3-mino-replication-replica-v2 (target) bucket and refresh, ensuring "Show versions" is enabled.

Observation: The specific version of beach.jpg that you permanently deleted in the origin bucket will not be deleted from the target bucket. Only delete markers are replicated, not permanent deletions of specific object versions.

Conclusion:

You have successfully set up and tested S3 Cross-Region Replication. You’ve learned that:

  • Versioning must be enabled on both source and destination buckets.
  • Replication rules apply to new objects by default; existing objects require a batch operation.
  • Object content and their version IDs are replicated.
  • Delete markers can be optionally replicated.
  • Permanent deletes of specific object versions are not replicated.
