Introduction

In this post, we’ll introduce Amazon Elastic Kubernetes Service (EKS), an alternative to Amazon ECS for running and managing containers on AWS. EKS leverages the power of Kubernetes, an open-source platform, to provide a robust solution for deploying, scaling, and managing containerized applications. Let’s dive into what makes EKS unique and how it fits into your AWS container strategy.

What is Amazon EKS?

Amazon EKS is a managed Kubernetes service that enables you to run Kubernetes clusters on AWS. Kubernetes, recognizable by its blue logo, is an open-source system designed for automating the deployment, scaling, and management of containerized applications — typically Docker containers. Unlike ECS, which is AWS-proprietary, Kubernetes is cloud-agnostic, supported by providers like Azure, Google Cloud, and others, offering a standardized API and ecosystem.

EKS vs. ECS

While both ECS and EKS aim to manage containers, they differ significantly:

  • ECS: A proprietary AWS service with a custom API.
  • EKS: Built on Kubernetes, providing an open-source foundation that supports multi-cloud portability.

Launch Modes in EKS

EKS supports two launch modes to suit different needs:

  • EC2 Launch Mode: Deploy worker nodes as EC2 instances, giving you control over the underlying infrastructure.
  • Fargate Mode: A serverless option where AWS manages the nodes, allowing you to focus solely on running containers without infrastructure maintenance.

Use Cases for EKS

EKS is ideal if:

  • Your company already uses Kubernetes on-premises or with another cloud provider and wants to maintain consistency.
  • You prefer the Kubernetes API and seek AWS-managed Kubernetes clusters.
  • You’re planning a multi-cloud strategy, as Kubernetes’ portability simplifies migrations.

EKS Architecture Overview

EKS operates within a VPC, typically spanning three Availability Zones (AZs) with public and private subnets. Here’s a breakdown:

  • EKS Worker Nodes: These are EC2 instances (or self-managed nodes) that host EKS Pods, the Kubernetes equivalent of ECS tasks.
  • Auto Scaling: Nodes can be part of an Auto Scaling Group (ASG) to handle demand dynamically.
  • Load Balancing: Expose EKS services using private or public load balancers (e.g., Application Load Balancer) for web access.

Node Types in EKS

EKS offers flexible node management options:

  • Managed Node Groups: AWS creates and manages EC2 instances within an ASG. Supports On-Demand and Spot Instances for cost optimization.
  • Self-Managed Nodes: You provision and register EC2 instances to the EKS cluster, offering customization. Use the pre-built Amazon EKS Optimized AMI or create your own AMI. Also supports On-Demand and Spot Instances.
  • Fargate Mode: No nodes to manage — run containers serverless-style with zero maintenance.

Storage Options for EKS

EKS supports attaching data volumes via a StorageClass manifest using the Container Storage Interface (CSI) compliant driver. Key storage options include:

  • Amazon EBS: Block storage for persistent data.
  • Amazon EFS: File storage, the only option compatible with Fargate.
  • Amazon FSx for Lustre: High-performance file system.
  • Amazon FSx for NetApp ONTAP: Enterprise-grade file storage.

These are critical keywords to remember for AWS exams.

Conclusion

Amazon EKS brings the power of Kubernetes to AWS, offering a managed, open-source solution for container orchestration. Whether you choose EC2 nodes for control, self-managed nodes for customization, or Fargate for simplicity, EKS provides flexibility to match your needs. Its storage integrations and multi-cloud compatibility make it a strong choice for modern architectures.
