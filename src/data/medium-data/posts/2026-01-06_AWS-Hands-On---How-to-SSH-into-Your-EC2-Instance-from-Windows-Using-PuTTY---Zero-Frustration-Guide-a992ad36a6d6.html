Introduction

If you’re on Windows and want to take full control of your Linux EC2 instance, PuTTY is the classic, free, and rock-solid way to do it. Even if you’re on Windows 10 or 11, this method works perfectly (and many pros still prefer it). In this step-by-step guide we’ll download PuTTY, convert your AWS .pem key into the .ppk format PuTTY loves, save a reusable session, and log in on the first try. Five minutes from now you’ll be typing commands inside your cloud server — let’s go!

What You’ll Need

  • Your running EC2 instance (Amazon Linux)
  • Public IP visible in the console
  • Port 22 (SSH) allowed in the security group
  • Your downloaded key pair file (.pem) — even if it has a space in the name, we’ll fix that

Step 1: Download & Install PuTTY

  1. Go to Link
  2. Download the 64-bit installer (top link)
  3. Run it → Next → Next → Install (takes 10 seconds)

You now have two important tools: PuTTY (the terminal) and PuTTYgen (the key converter).

Step 2: Convert Your .pem Key to .ppk (One-Time Only)

  1. Open PuTTYgen
  2. Click Load → change file type to “All Files (.)”
  3. Find your downloaded .pem file (e.g., EC2 Tutorial.pem) → Open
  4. Click Save private key → Yes (ignore passphrase warning)
  5. Save it as EC2Tutorial.ppk on your Desktop

Done — you now have a key PuTTY can use!

Step 3: Create a Re-Usable PuTTY Session

  1. Open PuTTY

2. In Host Name (or IP address) paste your instance’s public IP

3. Change the text to: ec2-user@YOUR_PUBLIC_IP (this auto-fills the username)

4. In the left menu: Connection → SSH → Auth

5. Click Browse → select your new EC2Tutorial.ppk file

6. Go back to Session at the top

7. In “Saved Sessions” type: My EC2 Instance

8. Click Save → now it’s there forever!

Step 4: Connect!

  1. Double-click your saved session (or select it + Load + Open)
  2. First time? Click Yes to accept the host key
  3. Boom — you’re logged in as ec2-user!

You’ll see the beautiful Amazon Linux banner and a prompt ready for commands.

Quick Test Commands

whoami          # → ec2-user
hostname        # → shows your instance name
ping google.com # → proves internet works (Ctrl+C to stop)

Pro Tips

  • Public IP changes when you stop/start the instance → just edit the saved session and re-save
  • Keep your .ppk file safe — never share it
  • Next time you want to connect: open PuTTY → double-click “My EC2 Instance” → you’re in!

Conclusion

You just turned your Windows laptop into a remote-control center for a real Linux server running in AWS — all with free tools and zero stress. PuTTY sessions are saved forever, so from now on connecting will literally be two clicks. You’re no longer just launching servers — you’re driving them. Next we’ll start installing real software and building something awesome inside your instance. Welcome to the cloud, Windows warrior! 🚀
