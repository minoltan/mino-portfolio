Introduction

Ready to feel the magic of the cloud? In this beginner-friendly hands-on guide, we’ll walk through launching your very first Amazon EC2 instance using the AWS Management Console. You’ll create a virtual server running Amazon Linux, automatically install a simple web server with a bootstrap script (User Data), access your “Hello World” webpage from anywhere, and learn how to stop, start, or terminate the instance to control costs. Best of all, we’ll stick to the AWS Free Tier so you won’t pay a cent (as long as you qualify and stay within limits). It takes just 10–15 minutes — let’s dive in!

Step 1: Access the EC2 Console and Start Launching

  1. Log in to your AWS Management Console (console.aws.amazon.com).
  2. Search for “EC2” in the top search bar and click on the EC2 service.
  3. In the left menu, click Instances.
  4. Click the orange button Launch instances.

Step 2: Name Your Instance

  • In the Name and tags section, enter “My First Instance” as the name. (This adds a helpful tag so you can easily find it later.)

Step 3: Choose an Operating System (AMI)

  • Under Amazon Machine Image (AMI), go to the Quick Start tab.
  • Select Amazon Linux.
  • Choose Amazon Linux 2 AMI (64-bit x86). It’s marked Free tier eligible — perfect for us!

Step 4: Select Instance Type

  • Under Instance type, keep the default t2.micro (or t3.micro if shown in your region). This is Free Tier eligible: 1 vCPU, 1 GB RAM, and you get 750 hours per month (enough to run it 24/7 for free).

Step 5: Create a Key Pair (for SSH Access)

  • Under Key pair (login), click Create new key pair.
  • Name: EC2 Tutorial
  • Key pair type: RSA
  • Private key file format: .pem (for Mac/Linux/Windows 10+) or .ppk (for older Windows with PuTTY).
  • Click Create key pair — it downloads automatically. Save it securely! (We’ll use this later for SSH, but not in this tutorial.)

Step 6: Configure Network Settings (Security Group)

  • Under Network settings, click Edit.
  • Your instance will automatically get a public IP (leave Auto-assign public IP as Enable).
  • In Firewall (security groups):
  1. Select Create a new security group.

2. It auto-adds SSH (port 22) from anywhere.

3. Check the box for HTTP (port 80) from the internet — this lets us view the web page.

  • Leave everything else default. (The security group will be named something like launch-wizard-1.)

Step 7: Configure Storage

  • Keep the default: 1 x 8 GiB gp2/gp3 root volume. Free Tier gives you up to 30 GiB of EBS storage — no changes needed.

Step 8: Add the Magic User Data Script (Bootstrap Your Web Server)

  • Scroll down to Advanced details (expand if needed).
  • In User data, paste this exact script:
#!/bin/bash
yum update -y
yum install httpd -y
systemctl start httpd
systemctl enable httpd
echo "<h1>Hello World from $(hostname -f)</h1>" > /var/www/html/index.html
  • This script runs automatically on first boot: updates the system, installs Apache web server, starts it, and creates a simple “Hello World” page.

Step 9: Review and Launch

In the right sidebar, confirm:

  • Number of instances: 1
  • Free Tier eligible items are highlighted.

Click Launch instance.

Click View all instances to go back to the dashboard.

Step 10: Wait and Check Your Running Instance

Refresh the page. Your instance will go from Pending to Running (takes 1–2 minutes).

Select your instance (checkbox) and note:

  • Public IPv4 address (e.g., 47.129.198.108)
  • Private IPv4 address (stays the same forever)
  • Security group with ports 22 and 80 open.

Step 11: Access Your Web Server

  • Copy the public IP.
  • Open a browser and go to: http://<your-public-ip> (make sure to include http://, not https!).
  • You should see: Hello World from [private IP] (If it loads slowly or not at first, wait 2–5 minutes and refresh — the script is still running.)

Step 12: Stop, Start, and Terminate (Manage Costs)

Select your instance.

Click Instance state (top menu):

  • Stop instance: Pauses it — no charges while stopped (but storage still uses Free Tier).
  • Wait for Stopped status, then Start instance to resume.

Important: When you stop and start, the public IP changes! Copy the new one and use http://new-ip.

To delete: Terminate instance (warning appears — data on the root volume is deleted by default).

Troubleshooting Tips

  • Web page not loading? Double-check HTTP (not HTTPS), security group has HTTP allowed, and wait 5 minutes.
  • No public IP? Edit network settings and enable auto-assign.
  • Free Tier warning: Always stop/terminate when done experimenting.

Conclusion

Congratulations — you just launched a real web server in the cloud in under 15 minutes, automated everything with User Data, and experienced the true power of AWS: spin up servers instantly, pay nothing while learning, and shut them down with one click. This is just the beginning of what EC2 can do! Practice stopping/starting a few times, then terminate the instance to clean up. In the next guide, we’ll connect via SSH and explore more. You’re now officially cloud-savvy — keep going! 🚀
