Introduction

Tired of downloading .pem files, fixing permissions, and typing long SSH commands? Meet EC2 Instance Connect — AWS’s brilliant browser-based terminal that lets you jump straight into your Linux instance with exactly two clicks. No key management, no terminal setup, works on any computer (Windows, Mac, Linux, even a tablet), and it’s 100% free. In this quick guide you’ll see exactly how it works, why it’s perfect for beginners (and pros who are lazy in the best way), and the one tiny security-group rule you must never forget.

How to Connect in 10 Seconds Flat

  1. Go to the EC2 console → Instances
  2. Check the box next to your instance (“My First Instance”)
  3. Click the big Connect button at the top
  4. Choose the tab EC2 Instance Connect
  5. Username is already filled as ec2-user (for Amazon Linux) — leave it!
  6. Click Connect

A new browser tab opens → boom, you’re inside your server! Type anything:

whoami          # → ec2-user
hostname
ping google.com

You now have a full terminal running directly on your EC2 instance, no SSH client required.

Why EC2 Instance Connect Wins Every Time

  • Zero key management — AWS uploads a temporary key for you
  • Works on any device with a browser — no Terminal/PowerShell/PuTTY needed
  • Perfect for learning, quick checks, or when you’re on a friend’s computer
  • Still uses real SSH under the hood, so everything you type is encrypted

The One Rule You Must Have (or It Won’t Work)

EC2 Instance Connect still needs port 22 open in your security group, exactly like normal SSH.

Required inbound rules:

  • Type: SSH
  • Protocol: TCP
  • Port: 22
  • Source: 0.0.0.0/0 (and ::/0 if you ever see IPv6 errors)

If you delete that rule → “There was a problem connecting” Add it back → instant success.
