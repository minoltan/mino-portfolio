Introduction

You’ve launched your EC2 instance and opened port 22 — now it’s time to actually log in and take control! SSH (Secure Shell) lets you open a terminal right inside your cloud server as if you were sitting in front of it. This quick, beginner-friendly guide walks you through every single click and command on Mac or Linux so you can connect in under 5 minutes — even if you’ve never touched a terminal before. Let’s turn that scary black screen into your new best friend!

What You Need (Checklist)

  • Your instance is running
  • It has a public IP
  • Security group allows SSH (port 22) from your IP or Anywhere (0.0.0.0/0)
  • Your .pem key file (the one you downloaded when creating the key pair)
  • The default user for Amazon Linux = ec2-user

Step 1: Prepare Your Key File

  1. Find the .pem file you downloaded (probably named something like EC2 Tutorial.pem).
  2. Rename it to remove spaces → EC2Tutorial.pem (spaces break things!)
  3. Move it to an easy folder, e.g., ~/aws-course/ (On Mac/Linux, ~ means your home folder)

Step 2: Open Terminal & Navigate to the Key

  1. Open Terminal (Mac: Spotlight → Terminal, Linux: Ctrl+Alt+T)
  2. Go to the folder containing your key:
cd ~/aws-course

3. Verify the file is there:

ls -l
# You should see EC2Tutorial.pem

Step 3: Get Your Instance’s Public IP

  1. Go to EC2 console → Instances
  2. Check your instance → Copy the Public IPv4 address (e.g., 13.214.177.82)

Step 4: Connect with SSH (The Magic Command)

First try (will probably fail — that’s normal!):

ssh ec2-user@13.214.177.82

You’ll get an error about authentication → we forgot the key!

Correct command:

ssh -i EC2Tutorial.pem ec2-user@13.214.177.82

If you see “Unprotected private key file” → fix permissions:

chmod 400 EC2Tutorial.pem

Now run the SSH command again:

ssh -i EC2Tutorial.pem ec2-user@13.214.177.82

First time? Type yes when it asks to trust the host.

Boom — You’re In!

You’ll see something like:

       __|  __|_  )
       _|  (     /   Amazon Linux 2 AMI
      ___|\___|___|

ec2-user@ip-172-31-xx-xx ~ $

Test It Out

whoami          # → ec2-user
hostname        # → shows the internal name
ping google.com # → proves internet works (Ctrl+C to stop)
exit # get out

Remember These Gotchas

  • Public IP changes when you stop/start the instance → copy the new one!
  • Always run chmod 400 on your .pem file (only needs to be done once)
  • If you get timeout → 99% chance port 22 is blocked in the security group
  • Keep your .pem file safe and never share it!

Conclusion

You just SSH’d into a real server running in a data center hundreds of miles away — using nothing but your laptop and a few lines of text. That’s the real power of the cloud! Save that command (maybe even make an alias), because from now on you’ll be jumping in and out of your instances like a pro.
