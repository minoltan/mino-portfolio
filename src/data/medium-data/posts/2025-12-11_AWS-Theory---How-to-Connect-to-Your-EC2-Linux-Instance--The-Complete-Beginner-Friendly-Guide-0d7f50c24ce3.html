Introduction

You’ve launched your first EC2 instance — awesome! Now comes the moment everyone gets a little nervous about: “How do I actually get inside this server?” Don’t worry — connecting to a Linux EC2 instance is easier than it looks, and AWS gives you several ways to do it depending on your computer. In this post we’ll cover every option (traditional SSH, PuTTY, Windows 10+ built-in SSH, and the super-simple browser-based EC2 Instance Connect) so you can pick the one that feels most comfortable. By the end, you’ll be logged in and ready to run commands like a pro — no stress required!

The Four Ways to Connect (Choose One!)

Quick Recommendation for Most People
Start with EC2 Instance Connect (browser-based). It works everywhere, needs zero setup, and is perfect while you’re learning. If you love the terminal later, switch to plain SSH — but you’ll never be stuck.

What You Need Before Connecting

  1. Your instance must be running
  • Think of your cloud computer (instance) like your home PC. You can’t use your PC unless you’ve turned it on.
  • What it means: The virtual server you want to connect to must be in the “Running” state. If it’s “Stopped” or “Terminated,” there’s no power and nothing to connect to.

2. It must have a public IP (or be reachable via VPC)

  • This is the address of your cloud computer on the internet. You need to know the address to send your connection request to it.
  • Public IP: This is a unique, external address (like a street address) that allows anyone on the internet to find and talk to your instance.
  • Reachable via VPC: If your instance is inside a private network (like an office network), it might not have a public address, but you would need a special path or connection (like a VPN) to reach it. For simple connection, a Public IP is what you usually need.

3. Port 22 (SSH) must be open in the security group (source: your IP or Anywhere for testing)

  • A computer has thousands of digital “doors,” called ports. Port 22 is the only door used for secure remote login (SSH). You must make sure that specific door is unlocked (open).
  • Security Group: This is a virtual firewall that controls traffic to and from your instance.
  • Port 22 (SSH): You must add a rule to the security group that specifically allows traffic on port 22.
  • Source: This specifies who is allowed to use this open door.
  • Your IP: The safest option. Only your specific computer is allowed to connect.
  • Anywhere (0.0.0.0/0): Allows any computer on the internet to attempt a connection. This is fine for quick testing but less secure.

4. You need the username: for Amazon Linux it’s always ec2-user

  • When you connect, you need to tell the computer who you are. This is your login name.
  • What it means: The default, main username for instances running the standard Amazon Linux operating system is fixed as ec2-user. You use this name, along with your secure key, to log in.

Method 1: EC2 Instance Connect (Easiest — My Personal Favorite)

  1. Go to EC2 → Instances
  2. Select your instance
  3. Click Connect (top-right)
  4. Choose EC2 Instance Connect
  5. Leave “User name” as ec2-user
  6. Click Connect → a terminal opens right in your browser!

Done. You’re in. No key files, no commands to type. Magic.

Method 2: Traditional SSH (Mac/Linux/Windows 10+)

  1. Make sure your key pair file (.pem) is downloaded and safe
  2. Open Terminal (Mac/Linux) or PowerShell/Command Prompt (Windows)
  3. Run this command (replace the key path and public IP):
ssh -i /path/to/your-key-pair.pem ec2-user@YOUR_PUBLIC_IP

First time? Type yes when it asks about the host key.

Method 3: PuTTY (Classic Windows)

  1. Download PuTTY from the official site
  2. Open PuTTY → Session
  • Host Name: ec2-user@YOUR_PUBLIC_IP

3. Go to Connection → SSH → Auth

  • Browse and select your .ppk file (convert .pem → .ppk using PuTTYgen if needed)

4. Click Open → login as: ec2-user

Common “It’s Not Working!” Fixes

  • Timeout / hanging → 99% chance port 22 is blocked in the security group
  • Permission denied (publickey) → wrong key file or wrong username
  • Connection refused → instance stopped or SSH service crashed (rare)

Pro tip: If one method fails, just try EC2 Instance Connect — it often magically works even when traditional SSH is being stubborn.

Conclusion

You only need ONE method to work — once you’re logged in, everything else in the course is the same. Whether you choose the slick browser terminal or old-school SSH, you now have the power to jump inside any Linux server in the cloud with just a couple of clicks.
